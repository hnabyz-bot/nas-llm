## 1. 검증 범위

전 구간(End-to-End)을 대상으로 한다:

```
NAS 원본 → 동기화 → raw/ 투입 → 인제스트 → wiki/ 생성 → 쿼리 → Git 커밋
```

## 4. 검증 절차

### 4.1 실행 순서

```
Phase A: 환경 (TC-01, TC-02)
    │
    ▼
Phase B: 핵심 기능 (TC-03 → TC-04 → TC-05 → TC-06 → TC-07 → TC-08 → TC-09)
    │
    ▼
Phase C: 인프라 (TC-10 → TC-11)
    │
    ▼
Phase D: 안정성 (TC-12)
```

### 4.2 판정 기준

| 등급 | 조건 |
|------|------|
| **PASS** | TC-01~TC-11 전 항목 통과 + TC-12 48시간 안정 |
| **CONDITIONAL** | TC-01~TC-09 통과, TC-10~TC-12 일부 미통과 (운영 조건부 승인) |
| **FAIL** | TC-01~TC-09 중 미통과 항목 존재 → 원인 분석 후 재시험 |

## 5. 결함 분류

| 심각도 | 설명 | 예시 |
|--------|------|------|
| Critical | 시스템 사용 불가 | 앱 미기동, 인제스트 전면 실패 |
| Major | 핵심 기능 장애 | 위키링크 미생성, 교차참조 실패, 모순 미감지 |
| Minor | 부가 기능 장애 | 그래프 뷰 렌더링 오류, auto-commit 타이밍 오차 |
| Cosmetic | 표시 문제 | frontmatter 포맷 불일치, 한국어 인코딩 깨짐 |
