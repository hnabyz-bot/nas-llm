2. Software Safety Classification
2.1 Classification Schemes
The manufacturer shall assign to each software system software safety class (A, B, or C)
according to the possible effects on the patient, operator, or other people resulting from a
HAZARD to which the software system can contribute.
2.2 Software Safety Classification
To concern the Classification of AspenView software, we refer to the document of the
“Guidance of the content of Premarket Submissions for Software contained in Medical
Devices” used in the FDA 510(k) and matched these results to the software safety
classification recorded in IEC 62304.
From the software safety classification in the EN62304:2006, the software safety classes
shall initially be assigned based on severity as follow:
Class A : No injury or damage to health is possible.
Class B : Non-SERIOUS INJURY is possible.
Class C : Death or SERIOUS INJURY is possible.
The result of comparison between FDA 510(k) and IEC 62304. We’d like to define the
classification the followings: Class A is Minor, Class B is Moderate and Class C is Major.
■ Determining Level of Concern
If the answer to any one question below is “Yes”, the Classification of the Software Device is
likely to be Moderate.

2.3 Conclusion
AspenView software consists of five modules (Worklist, Exam, Browser, Viewer, Setting).
Each module we checked the classification based on the “Question Tale”. And, Checked the
output of the risk management process, the AspenView would be made the highest risk like
“Tissue damage (X-ray burn) Cancer (DNA modification)” when the risk could arise from a
failure of the software system to behave as specified, and the probability of such failure shall
be assumed to be 100 percent. So, we assumed that this risk can be resulted in Non-Serious
Injury. And, the result of checking the each module is proved the ‘Worklist’ Module and is
Class A (no injury or damage to health is possible). And ‘Exam, Browser, Viewer, Database,
DICOM’ module are Class B (Non-serious injury is possible) based on the output of the risk
management process.
Therefore, based on the result of risk management, AspenView is classified as the Class B,
Moderate.
