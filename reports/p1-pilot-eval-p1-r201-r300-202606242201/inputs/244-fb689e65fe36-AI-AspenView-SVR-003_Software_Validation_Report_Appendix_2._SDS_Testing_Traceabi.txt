Software Design Specification (SDS)
1. Introduction
Purpose of this document is to describe relations between SRS, source code and
Operation Manual. This document enumerates (enlists, formalizes) items of SRS and
Operation Manual for traceability.
2. Development Methods and Life Cycle Model
Software development process follows classic methodology of Waterfall Life Cycle Model, as
illustrated in the following chart.
2.1 Definition
1) Analysis
The scope of software development is defined and analyzed. The activity consists of establishing
requirements and work scope before the design phase. The requirements are documented for
each level of the software-specific project and design notes. When enough of the requirements
are gathered and analyzed for the iteration of the prototype, the activities shift into the design
phase.
2) Planning
Specific step by step activities for projected work scopes are planned. The tasks, responsibilities,
resources, and other aspects pertinent to the specific development project are outlined.
2.2 Design
The design phase addresses four functional attributes of the software: data structure, software
architecture, functionality, and interface characterization. Each attribute is designed and
documented.
2.3 Implementation
The implementation phase converts the design into a machine language. The software source
code is created in accordance with coding conventions. Design reviews/inspections are held to
verify the consistency of the code created with respect to the company standards.

2.4 Qualification.
The qualification phase verifies that the design and code implemented meet the original
requirements. Testing ensures that defined inputs will produce intended results that coincide with
required outputs.
2.5 Maintenance.
Once the software is released, the only software changes allowed are to fix Identified defects in
the code or to support approved enhancements. All identified system software defects are
recorded on System Discrepancy Reports (SDR) forms or submitted electronically to the
Modification Request (MR) database

3.3 Image Processing
1) Directional smooth
A. Theoretical basis : Directional smooth technique is based on the theory of anisotropic
diffusion. This method is an extension of the technique proposed by Perona and Malik
(1990), which analyzes the directionality of the image using the local structure tensor.
B. Mathematical model :
The local structure tensor J is defined as:
J = [J11 J12; J12 J22] = [∇I ∇I^T]
where ∇I is the gradient of the image. The directional smoothing is modeled by the
following partial differential equation:
∂I/∂t = div(D∇I)
where D is the diffusion tensor, which is constructed based on the eigenvectors of the
structure tensor J.
C. Algorithm Details :
(1) Initial noise reduction using Gaussian filter
(2) Compute structural tensor and eigenvalue decomposition
(3) Construct diffusion tensor D
(4) Solve diffusion equation using iterative numerical solution
D. Optimization and Implementation
(1) Parallel processing using SIMD (Single Instruction, Multiple Data) instructions
(2) Increased calculation speed through GPU acceleration
(3) Large-scale image processing using multithreading
E. Clinical significance
Directional smooth technique is particularly important in medical images that contain
highly directional anatomical structures such as vascular structures or fibrous tissues.
This technique effectively removes noise while preserving important directional
information, thus improving diagnostic accuracy.
2) Adaptive Unsharpened with Gaussian / Laplacian pyramid
A. Theoretical basis : This technique combines the image pyramid concept proposed by
Burt and Adelson (1983) with the adaptive histogram equalization theory of Pizer et al.
(1987). The multi-scale representation allows for individual processing of different
frequency components of the image.
B. Mathematical model :
Each level l of the Gaussian pyramid is defined as:
G_l = downsample(G_(l-1) * g)
where g is the Gaussian kernel.
Each level of the Laplacian pyramid is computed as:
L_l = G_l - upsample(G_(l+1))
Unsharp masking is expressed as:
S_l = G_l + k * (G_l - G_l * b)
where k is the sharpening strength and b is the blur kernel.
C. Algorithm Details :
(1) Generate Gaussian pyramid for input image
(2) Compute Laplacian pyramid
(3) Measure local contrast at each pyramid level

(5) Reassemble modified Laplacian pyramid to generate final image
D. Optimization and Implementation
(1) Optimization of large-scale convolution operations using FFT (Fast Fourier

(2) Minimization of memory access through cache-friendly algorithm design
(3) Implementation of heterogeneous computing using OpenCL
E. Clinical significance : This technique is particularly useful in complex medical images

maintaining overall image quality, facilitating the detection of microcalcifications or small

3) Protective contours
A. Theoretical basis : This technique is particularly useful in complex medical images
containing structures of varying sizes. It improves the visibility of fine structures while
maintaining overall image quality, facilitating the detection of microcalcifications or
small tumors.
B. Mathematical model :
The contour C is defined as the curve
minimizing the following energy function:
E(C) = ∫ (α|C'(s)|^2 + β|C''(s)|^2 + λE_image(C(s))) ds
where α, β, λ are the weights, and E_image is the image-based energy term.
The protection mask M is generated as follows:
M(x, y) = exp(-d(x, y)^2 / (2σ^2))
where d(x, y) is the distance from pixel (x, y) to the nearest contour.
C. Algorithm Details :
(1) Initial contour setting (automatic or semi-automatic)
(2) Gradient vector flow (GVF) calculation
(3) Iterative contour optimization using active contour model
(4) Protection mask generation based on final contour
D. Optimization and Implementation
(1) Improving computational efficiency using narrow-band level set method
(2) Parallel implementation of contour evolution
(3) Optimizing memory usage through adaptive mesh technique
E. Clinical significance : Protective contours technique is important for preserving
important anatomical boundaries such as organ boundaries or tumor margins. This
enables accurate segmentation and measurement, especially in computer-aided
diagnosis (CAD) systems, thus improving diagnostic accuracy.
4) Median Filter
A. Theoretical basis : Median Filter is a nonlinear digital filtering technique that is mainly
effective in removing 'salt-and-pepper' noise. This filter is widely used in medical imaging
because it can remove noise while preserving edges.
B. Mathematical model :
The output y[m,n] of the median filter is defined as:
y[m,n] = median{x[i,j], (i,j) ∈ w}
where w represents the filter window.
C. Algorithm Details :
(1) Apply a sliding window to the input image
(2) Sort the pixel values within the window

(4) Repeat for the entire image
D. Optimization and Implementation
(1) Speed improvement through histogram-based median calculation
(2) Optimization of large-scale image processing through parallel processing
E. Clinical significance : Median Filter effectively removes impulse noise from medical

5) Sharpen
A. Theoretical basis : Sharpen technique enhances the high frequency components of an
image to make edges and details more distinct. It is mainly based on the principle of
Unsharp Masking.
B. Mathematical model :
The sharpened image S is computed as:
S = I + λ(I - I_blur)
where I is the original image, I_blur is the blurred image, and λ is the sharpening
strength.
C. Algorithm Details :
(1) Apply Gaussian blur to the original image
(2) Calculate the difference between the original image and the blurred image
(3) Multiply the difference by the intensity coefficient and add it to the original image
D. Optimization and Implementation
(1) Optimization of blur operation using separable Gaussian filter
(2) Parallel processing using SIMD instructions
E. Clinical significance : The Sharpen technique emphasizes the fine structures of X-ray
or MRI images, making it easier to identify small lesions or fractures.
6) Contrast
A. Theoretical basis : Contrast enhancement is a technique that expands the dynamic
range of an image to increase visual distinction. Methods such as histogram
equalization and contrast stretching are used.
B. Mathematical model :
For contrast stretching, the new pixel value p_new is computed as follows:
p_new = (p - min) * (newMax - newMin) / (max - min) + newMin
Where p is the original pixel value, min and max are the minimum and maximum
values of the original image, and newMin and newMax are the new range.
C. Algorithm Details :
(1) Analyze image histogram
(2) Determine contrast enhancement function (linear or non-linear)
(3) Create lookup table
(4) Map pixel values
D. Optimization and Implementation
(1) Fast pixel value conversion using lookup tables
(2) Parallelization of histogram calculations
E. Clinical significance : Contrast enhancement improves diagnostic accuracy by making
it easier to distinguish between tissues in low-contrast medical images (e.g., chest X-
rays).
7) Edge Enhancement
A. Theoretical basis : Edge Enhancement is a technique that emphasizes the edges of an
image to make the boundaries of structures clearer. It usually uses a high-pass filter or
a Laplacian filter.
B. Mathematical model :

Edge enhancement using the Laplacian filter is expressed as:
I_enhanced = I + λ * ∇²I
where ∇²I is the Laplacian of the image, and λ is the enhancement strength.
C. Algorithm Details :
(1) Apply edge detection filter (e.g. Sobel, Prewitt)
(2) Calculate the intensity of the detected edge
(3) Add weighted edge information to the original image
D. Optimization and Implementation
(1) Reduced computational complexity by using separable filters
(2) Parallel processing using GPUs
E. Clinical significance : Edge Enhancement improves diagnostic accuracy by clarifying

8) Gamma Correction
A. Theoretical basis : Gamma Correction is a technique that nonlinearly adjusts the
luminance or trichromatic component values of an image. It is used to compensate for
nonlinearities in display devices or to adjust the overall brightness of an image.
B. Mathematical model :
The gamma-corrected pixel value V_out is calculated as:
V_out = A * V_in^γ
where A is a constant, V_in is the input pixel value, and γ is the gamma value.
C. Algorithm Details :
(1) Normalize pixel values of input image to 0-1 range
(2) Apply gamma function to each pixel
(3) Rescale result value to original pixel value range
D. Optimization and Implementation
(1) Fast gamma correction using lookup tables
(2) Parallel processing using SIMD instructions
E. Clinical significance : Gamma correction improves tissue contrast in X-ray images,
making it easier to distinguish between soft and hard tissues.
