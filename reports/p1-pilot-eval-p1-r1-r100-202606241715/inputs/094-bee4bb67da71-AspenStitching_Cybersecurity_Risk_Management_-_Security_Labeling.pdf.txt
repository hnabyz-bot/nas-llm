2. Device Instructions & Product Specifications
2.1 General Product Description
• AspenStitching is an advanced medical imaging software designed specifically for
automating the process of stitching long-length images generated by radiological
devices such as CR (Computed Radiography) and DR (Digital Radiography) systems.
The primary function of AspenStitching is to enhance the diagnostic capabilities of
healthcare providers by enabling seamless integration and processing of multiple image
files into a single, cohesive image.
• Note that AspenStitching is intended to be used in conjunction with AspenView
software in order to effectively combine the X-ray images, which can only be taken in
parts of the body. AspenStitching will take these partial images and produce a
comprehensive view of the patient’s X-ray
2.2 Purpose of the Software
The AspenStitching software serves several critical purposes in a clinical setting:
• Image Stitching: It effectively merges multiple DICOM, TIFF, or raw images taken from
radiological equipment into a unified image, allowing for comprehensive views of the
anatomy being examined. This is particularly useful in procedures requiring long-length
images, such as imaging larger areas of the body.
• DICOM Compatibility: AspenStitching adheres to the DICOM (Digital Imaging and
Communications in Medicine) standard, facilitating interoperability with various imaging
systems and PACS (Picture Archiving and Communication Systems). This ensures that
images can be easily transferred, stored, and retrieved within healthcare environments.
• User-Friendly Interface: The software features an intuitive interface that allows
radiologists and technicians to manage image inputs efficiently. Users can manually or
automatically configure the stitching process, adjusting parameters such as overlap and
cropping to ensure optimal image quality.
• Configuration Flexibility: AspenStitching allows users to customize settings based on
the specific requirements of each imaging task. This includes adjusting for different
types of exposures (e.g., single exposure vs. multi-exposure), image types, and output
formats.

2.8 Operating principle
• A radiograph is an X-ray image obtained by placing a part of the patient in front of an X-
ray detector and then illuminating it with a short X-ray pulse. To take an X-ray image,
the operator sets the patient in front of the detector and sets the parameter values of
the high voltage generator to control the intensity of X-ray for each body part. The
detector changes the X-rays coming through the part of the patient to digital signals and
makes digital images.
• An Acquisition program is used to acquire, process, storage and view the taken X-ray
image by detector or Image plate. After exposure and readout, the raw imaging data
must be sent and processed for display on the computer. Image processing is one of
the key features of digital radiography, greatly influencing the way the image appears to
the radiologist. Image processing is used to improve image quality by reducing noise,
removing technical artifacts, and optimizing contrast for viewing.

3.2 Manufacturer Responsibility - Network Security
• AspenStitching uses secure communication protocols (e.g., HTTPS, SSL/TLS)

the manufacturer.

incidents immediately.

• Avoid connecting the device to untrusted networks or devices.

4. Secure Configuration of AspenStitching
4.1 Initial Secure Configuration
Endpoint Protections
• Anti-Malware: Ensure that reputable anti-malware software is installed
and up-to-date. Configure it for real-time protection and regular scans.
o Instruction: Install anti-malware software, configure it to automatically
update its virus definitions, and schedule weekly full system scans.
• Firewall: Configure firewall settings to allow only necessary traffic for
AspenStitching operations. Define rules to block unauthorized inbound and
outbound connections.
o Instruction:
▪ Inbound Rules: Allow TCP connections on the port
AspenStitching uses for communication (consult AspenStitching
documentation for the specific port number).
▪ Outbound Rules: Allow connections necessary for
AspenStitching functionality (e.g., updates or license validation).
Physical Security Detection
• Physical Security: Ensure the physical security of the hardware running
AspenStitching. This includes securing access to the server or workstation
and monitoring for unauthorized physical access.

4.2 User-Configurable Changes and Associated Risks
User-Configurable Changes
• Network Configuration: Users might modify network settings, such as
firewall rules or network segmentation.
o Risk: Inadequate network security could expose AspenStitching to
unauthorized access or attacks.
o Recommendation: Users should only adjust network settings under
guidance from IT security professionals.
• Software Updates: Users can choose to enable or delay software
updates.
o Risk: Delayed updates can leave AspenStitching vulnerable to known
security threats.
o Recommendation: Enable automatic updates for AspenStitching to
ensure security patches are applied in a timely manner.
• Log Management: Users can alter log settings, including log retention and
log level.
o Risk: Inadequate logging might lead to missed detection of security
incidents.
o Recommendation: Follow standard logging practices and consult with
IT security to configure appropriate log levels and retention periods.
• Access Controls: Users can modify access control settings for different
roles and permissions.
o Risk: Misconfigured access controls might allow unauthorized access to
sensitive features or data.
o Recommendation: Regularly review and audit access control settings
to ensure that permissions are correctly assigned.

4.3 Changes that May Increase Security Risk
• Disabling Antivirus/Anti-Malware: Turning off these protections exposes
the system to malicious threats.
o Avoid: Always keep antivirus/anti-malware solutions active.
• Opening Unnecessary Ports: Allowing additional network traffic can
increase exposure to vulnerabilities.
o Avoid: Only open ports required for AspenStitching’s operation and
regularly review open ports.
• Weak Passwords: Using simple or default passwords can be easily
compromised.
o Avoid: Enforce strong password policies and regular password
changes.
• Ignoring Updates: Skipping updates can lead to unpatched vulnerabilities.
o Avoid: Keep AspenStitching and all related software up-to-date with the
latest security patches.
