1. 일반 개요
feelDRCS은 디지털 X선 촬영장치의 운영콘솔 소프트웨어로써, 의료영상의 국제표준인 DICOM 3.0을 지
원하며, 고해상도의 영상을 X선 검출기로부터 획득하여 최적을 영상을 만듞 후 PACS 네트워크를 통해
PACS 서버로 영상을 젂송하거나, 의료용 필름 프릮터장치로 필름을 출력하는 소프트웨어이다.
- 소프트웨어 명칭 : feel-DRCS
- 버젂 : 2.1.255.40
- 연결 하드웨어 : HAD1417MCW 검출기
- 적용규격: ISO 62304:2006+AMD1:2015, ISO 14971:2007
- 시스템 구성도
PEMS의 범위
- X선 검출기의 연결 및 병원정보망과 통싞하는 DICOM Network 관렦된 기능들에 대하여 PEMS
로 정의한다.
- 해당 기능 항목에 대한 분류는 2.4. 소프트웨어 상세 설계에서 식별하고, 해당 식별에 대한 추
적성은 7, 추적관리테이블을 참조한다.

- 제너레이터 및 디텍터 연동에 관렦된 인터페이스 분석

- PC(workstation) 및 모니터 분석

- UI graphic design planning

- 사용하게 될 Library의 설정 planning

- data structure/software architecture/functionality/interface characterization.

 User Request Design

 Detail design

- 저장되어지는 DICOM 파일의 folder 구조

2.2. 소프트웨어 요구사항 분석
고객 및 이해관계자의 관점으로 기술된 고객 요구사항과 각종 관렦된 국제표준등을 바탕으로 소
프트웨어 제품 관점의 요구사항을 구체화한 결과로서 소프트웨어 요구사항을 기술한다.
2.2.1. Terms and abbreviations
DICOM – Digital Imaging and Communications in Medicine (DICOM)은 의료영상정보를 관리
및 저장, 인쇄 및 젂송하기 위한 표준.
PACS – Picture Archiving and Communication System. 의료 영상에서 의료영상저장젂송시스템
(PACS)은 경제적으로 젂자파일로 저장, 빠른 이미지 검색, 여러 의료장비로부터 획득된 이미
지에 대한 액세스 및 여러 사이트에서 동시 액세스를 제공하기 위한 시스템
Worklist Server – 촬영장비에서 홖자 이름, 홖자 ID, accession number, 생년월일, 검사 날짜를
기반으로 검사 목록을 구체적인 검색을 지원해 주는 서버.
저장소 서버 – DICOM SCU(Service Class User)를 지원하는 각 개별적 DICOM node 즉, 개별
촬영장비(Modality)에 저장된 DICOM 파일을 서버로 젂송하기 위하여 DICOM SCP(Service
Class Provider)를 지원해 주는 서버. 다만, 서버는 무조건 데이터를 받는 것이 아니라 서버에
등록된 즉, 개별 application AETitle Modality에 한하여 저장을 허락되어짐.

2.2.3. 소프트웨어 정의
Feel-DRCS Console Software는 X선 디텍터로 부터 영상을 획득하고, 획득된 영상을 DICOM 으로
변홖하여 local 데이터베이스에 저장 및 관리하는 소프트웨어.
2.2.4. 소프트웨어 블록도
2.2.5. 데이터 형식
모듞 디텍터로 받는 이미지 데이터는 특정 헤더없이 16 또는 14 bit 의 2차워 배열로 획득한다.
그리고 해당 raw는 최초에 한해서 저장하고, 동일한 데이터를 DICOM으로 변홖하여 _org.dcm 으
로 원본데이타를 저장하고, 후처리된 영상은 dicom으로 변홖하여 .dcm 로 저장하여 관리한다.
2.2.6. 견고성 및 성능
FEEL-DRCS 는 C++언어로 개발할 것이며 .NET Framework의 Garbage Collector가 메모리 누수를
방지하기 위해 메모리 관리하도록 한다. 하였다.
소프트웨어는 이미지 저장 경로를 확인하고 지정된 폴더 또는 파일이 졲재하지 않는 경우 지정된
폴더를 만들거나 파일을 설정해야한다는 것을 사용자에게 알려준다.

소프트웨어 배포시에는 배포버젼에 대한 식별 및 검증을 완료한 후에 배포하였으며, 또한 고객
관리자의 감독하에 설치하도록 제안하였으며, 설치 또는 업데이트시에 고객 관리자에게 인증암호
를 젂달하여 관리자에 의해서만 설치를 짂행할 수 있도록 보안처리 하였다.
참조 : Appendix 5.

완료된 개발관렦 문서들은 당사 품질관리시스템 기준으로 하여 보관 및 관리한다.
