Introduction

feel-DRCS 는 imfoU 사의 Digital Radiography Control Software의 제품명 입니다.

feel-DRCS는 flat-panel 또는 CCD 등의 DR detector로부터 영상 획득 및 임상적용을 위한 빠른 속도의 영상처리, 그리고 PACS 연동을 위한 DICOM 서비스를 제공하는 DR console 또는 DR control software 입니다.

이 매뉴얼은 feel-DRCS 소프트웨어를 installation 및 operation 하기 위한 내용을 정리되어 있습니다.

IMFOU CO.,LTD.

#1105, Daerung PostTower7, 48, Digital-ro 33-gil, Guro-gu, Seoul, 08377. Korea.

Purpose

feel-DRCS 는 FS-MLW(Faster Specialized Multi Layered Wavelet) 기술을 기반하여, 각종 detector나 X-Ray Generator와 integrated 된 Operation Console System 소프트웨어입니다.

feel-DRCS 는 절대적인 사용자 입장에서 개발된 제품으로, 다년간 임상사이트인 병원에서 디자인 되었으며, 서비스를 위한 DR 제조사들의 서비스 엔지니어들에 의해 완성된 제품입니다. 그러므로 쉽고, 간결하면서도 다양한 기능들이 포함된 최상의 User interface로 개발된 DR Console system입니다.

feel-DRCS 는 feel-TK를 기반으로 내장되어 있기 때문에 보다 빠른 속도로 노이즈와 각종 artifact 제거되고, 보다 선명한 생체조직을 볼 수 있습니다. 그리고 독자적인 calibration 기능이 내장되어, 모두 종류의 detector에 대한 기본 calibration data(offset image and gain image)을 설정할 수 있습니다. 그리고 defect view를 제공하여 진행성 defect들을 처리할 수 있습니다.

feel-DRCS 는 전세계적으로 판매되고 있는 각종 detector들에 대해서 이미 소프트웨어 검증 및 판매가 되어서 임상필드에서 사용되고 있습니다. DR제조사가 사용하고자 하는 detector를 손쉽게 설정에서 타입만 변경하면 바로 콘솔시스템의 소프트웨어로 사용이 가능합니다. 그리고 통합 DR 시스템인 X-ray 의 generator operation panel을 소프트웨어 내부에 integration 되어 있어, APR, AEC, Ready/Exposure등을 제어가 손쉽게 가능합니다.

feel-DRCS 는 각종 X-ray 제너레이터와 연동이 되어, APR 기능 등을 지원하여, 하나의 콘솔 소프트웨어에서 기존의 Operation Panel의 역할을 같이 수행할 수 있게 하여 분리된 DR제품이 아닌 하나의 통합된 DR제품을 지원합니다.

feel-DRCS 는 단순히 소프트웨어를 공급하는 것이 아니라, 다년간 경험했던 노하우를 DR 제조사에게 전달해 드리며, 동시에 컨설팅까지 같이 지원해 줌으로써 조력자(implement for you)의 역할을 다 할 것입니다.

2-1. 사용용도

Feel-DRCS는 방사선사 및 의사들이 일반 X선 영상을 획득하기 위한 용도로 사용된다.그래서 병원에서 근무하는 의사 및 방사선사외에는 사용할 수 없다. 또한 이 소프트웨어는 서비스 엔지니어에 의해서만 설치되고 방사선기사 및 의사에 의해서만 사용되어진다.

For proper usage and safety purposes, please ensure that only legally qualified person (Doctor, Medical Treatment Radiological Technologist, Medical Treatment X-RAY Technologist) can use this product after fully familiarize with contents of user manual

This medical device can’t use alone. So this product need to connect with X-ray detector. Sometimes, this software operates with exposure condition of X-ray Generator system.

This product is a medical device, and it should only be used as digital X-ray radiography console software for X-ray radiography at corresponding hospitals.

저장된 영상을 오픈하여 편집을 하거나 PACS 서버로 전송, Print로 출력할수 있다.

Review

Worklist에서 선택한 정보로 촬영을 하거나 Browser에서 저장되어 있는 영상을 불러와 편집할 수 있다. Generator를 제어하기 위해서 Generator탭과 영상을 편집하기 위해서 Process탭이 있다.

Config

Option Mode

장비의 구성에 따른 추가적인 view 모드들이 제공됩니다.Viewsub (Generator Console 지원 모니터 viewer)

feelView(의료용 모니터를 추가하여 영상확인하고자 하는 경우)

Operation feel-DRCS

Login

프로그램을 시작한다.

바탕화면에서 feel-DRCS 단축아이콘을 더블 클릭하여 프로그램을 실행한다.

Login 화면에 사용자 이름(혹은 아이디)과 비밀번호를 입력하고 OK 버튼을 선택하여 프로그램을 시작한다.

로그인을 하고 나면 최초에 각종 연결된 detector 및 generator 와의 연결을 확인하여 화면의 각 부위의 연결된 color 값을 입력하게 된다.

- Company 또는 회사이름, SUPERVISIOR등의 사용자 별에 따라 기능 권한이 차이가 나게 된다.

- 일반사용자 및 일반관리자는 병원에서 사용가능 한 계정이며, SUPERVISIOR 계정은 A/S 엔지니어 사용자 계정임.

- 로그인 후 장비의 연동에 따라 잠시 화면이 멈춰 있게 보일 수도 있다

 Worklist

설정된 Worklist서버에서 Study를 읽어와 화면에 표시한다. 수동으로 입력할 수도 있다.

리스트에서 환자정보를 더블 클릭하거나 Exam버튼을 클릭하여 촬영모드로 이동할 수 있다.

Main Menu

EXAM : 아래 리스트에서 선택된 검사에 대해서 촬영모드로 전환

DELETE : 선택된 촬영 검사 목록을 삭제한다.

REFRESH : DICOM Worklist 가 지원되는 경우에 목록 갱신한다.

ER : 응급촬영을 실시한다.

CODE : RIS 코드를 추가 및 수정하기 위하여 code manager 콜한다

ER 의 경우에 사전에 코드 매니저에서 ER001 special code를 사전에 설정해 두어야 아이디, 이름등 자동으로 생성되면서 촬영 모드로 넘어가게 된다. 이때 검사코드에 촬영부위, 촬영조건등을 기입해 놓은 것이 촬영설정이 되어지게 된다.

Search option

리스트에 표시되는 항목을 설정할 수 있다.

환자정보(ID, Name, Access No.)를 이용하여 특정환자만 표시하거나 Schedule, Modality, State, D.O.B(Date Of Birth) 항목을 이용하여 특정 조건에 맞는 환자만 표시할 수 있다.

Refresh condition

워크리스트 서버에 연결이 가능할 경우 서버에서 환자정보를 가져올 주기를 설정할 수 있다.

AUTO 가 보이게 되는 경우에는 우측에 설정된 시간 주기로 자동으로 리스트를 갱신하게 된다.

만약 자동 리스트 갱신을 원하지 않는 경우에는 AUTO 버튼을 눌러서 MANUAL로 보이게 하면 원하는 경우에만 refresh 하게 된다. 이때 refresh 버튼은 6-2-1. 에 있는 REFRESH 버튼을 클릭하여 수동 목록 갱신을 할 수 있다.

New SET

 각종 설정(Body part, View position, X-ray exposure condition, Image processing parameter, Annotation etc)을 수정하거나 추가하기 위하여 NewSET을 콜 한다. 사용법은 Generator 부분을 참조한다.

List

Option에 설정된 조건에 따라 해당되는 목록이 표시된다.

Worklist 목록에서 마우스 오른쪽 버튼을 클릭하면 Pop-up 메뉴가 나타난다.

Reserve (X) : 선택한 목록을 예외적인 이유로 인하여 상태를 촬영보류상태(Reserve)로 변경한다.

Show : 선택한 검사 상황을 만족하는 검사 목록을 검색한다.

All : 검색 조건 없이 모든 검사 목록을 표시한다.

R : 검사하지 않은 목록을 검색한다.

X : 촬영보류상태로 강제 변경시킨 목록을 검색한다.

Y : 검사가 종료된 목록을 검색한다.

Search Option : Worklist 목록 검색 조건을 표시하거나 숨긴다.

Code Manager… : 선택한 검사의 RIS code에 대해 View position을 선택하는 Code Manager를 화면에 표시한다. 사용법은 “8. TOOLS의 code manager” 를 참조한다.

Export to Excel : Exam list에 보여지는 환자 목록을 Excel 파일로 내보낸다. (이 기능이 올바르게 동작하기 위해선 feel-DRCS 폴더 내 WorkInfo.xls 파일이 존재해야 한다.)

Phantom : Phantom를 촬영하기 위한 옵션으로 영상 획득 후 Parameter 적용이 되지 않는다. (관리자 SUPERVIOR, COMPANY 등 관리자 권한으로 로그인시 나타난다.)

Manual Input

임의로 환자정보를 입력하여 새로운 검사 정보를 만들수 있다.

Manual Input

ID, Name, ACC. No. 등을 입력한다. 최소한 ID, Name을 입력해야 검사를 실시할 수 있다. 만약 한번 이라도 촬영 정보를 입력한 경우가 있다면 해당 ID를 입력하고 Enter 키를 누르면 해당 ID의 나머지 환자정보가 자동으로 입력된다. 그리고 각 항목 이동 시 TAB 키 사용이 가능하다.

Anatomy의 선택은 아래의 그림처럼 그림에서 선택할 수 있다.

만약 사전에 자주 사용하는 검사목록을 special code에 입력된 상태라면 special code 옆의 코드를 선택하면 자동으로 study list에 검사목록이 확인된다.

Worklist 창에서 Special code를 수정 또는 새로이 입력하고자 하는 경우에는 EXAM list 위에서 우측 버튼을 클릭하여 Code Manager 실행한다.

D.O.B(Date of Birth,생년월일)를 입력하면 DICOM 파일에 생년월일과 나이가 입력된다.

Date of Birth를 1900.01.01로 그대로 두면 DICOM 파일의 Patient’s Birth Date 필드(0010,0030)에 값이 입력되지 않는다.

Pat Age(나이)를 입력하면 DICOM file에 나이만 입력된다.

나이를 Y(연 단위), M(개월 단위), D(날짜 단위)로 지정할 수 있다.

Description에 Study Description를 임의로 입력할 수 있다.

Date of Birth(생년월일)과 Pat Age(나이)를 따로 입력하게 되면, DICOM 파일에 입력한 대로 저장이 되어 생년월일과 나이가 다르게 입력될 수 있으므로 주의해야 합니다.

Study List

ADD : 입력된 검사정보에 대하여 위의 좌측그림에 선택 한 검사목록에 추가한다.

DEL : 잘못 추가된 검사목록을 삭제한다.

UP/DOWN : 선택한 검사목록의 순서를 수정하고자 할 때 이 버튼을 이용하여 순서를 수정한다.

OK : 입력한 검사 정보를 검사목록(EXAM List)에 추가한다.

CANCLE : 입력한 검사정보를 초기화 한다.

EXAM : 입력된 검사내용으로 바로 검사를 실시하기 위한 촬영모드로 전환한다.

 Browser

Browser 모드는 현재 저장되어 있는 영상데이타를 review 하기 위하여 영상을 선택하는 database viewer의 역할을 한다.

이 모드에서는 다양한 방법으로 검색도 가능하면 추가촬영, image 순서 바꾸기, PACS 로 전송 또는 백업 및 restore 가 가능하기 때문에 영상관리에 매우 유용한 모드로 사용이 가능하다.

Browser mode를 실행하면 항상 초기에는 image display 목록에서는 영상이 보이지 않는다. 그 이유는 Patient/study/series의 목록에서 선택이 되어 있지 않기 때문이다. 그러므로 항상 영상을 보이도록 하기 위해서는 보고자 하는 목록리스트에서 선택을 하게 되면 영상이 나타나게 된다

Main Menu

OEPN : 선택된 환자의 영상을 모두 오픈하여 보여준다.

EXPOSURE : 선택된 검사에 대하여 다시 재촬영을 할 수 있도록 촬영모드로 전환한다.

MERGE : 서로 다른 Study를 합칠 때 사용한다. 예를 들어 같은 환자의 영상중에 하나가 잘못되어 다른 환자로 입력되어 촬영된 경우, 또는 한 환자의 영상을 여러 개의 응급촬영을 한 경우에 merge 하기 위하여 사용한다.

우선 합쳐지고자 하는 환자를 선택한다.

그런 다음 MERGE 버튼을 클릭한다.(main menu에서)

MERGE 버튼을 클릭하면 위와 같은 팝업창이 뜬다.

조건 또는 전부 보이는 환자 목록 중에서 합치고자 하는 환자를 선택한다.

선택된 환자의 아래에 표시된 검사 목록 중에서 합치고자 하는 목록을 선택한 후에 Merge 버튼을 눌러서 검사를 merge 한다.

검사 목록(Study) 모두 merge된다.

ANALYSIS : 최초 설치시에 Analysis가 체크된 이후부터 촬영된 영상들에 대한 전반적인 분석표를 볼 수 있다.

 EDIT

아래의 EDIT 버튼은 아래의 그림처럼 검사정보를 수정하기 위하여 사용된다. 이때 누가 수정한 것인지 확인하기 위하여 계정확인을 실시한다.

 DEL

아래의 DEL 버튼은 삭제하고자 하는 검사를 선택하여 삭제한다.

이때에도 마찬가지로 계정확인을 실시한다.

Thumbnail

선택된 검사에 대한 thumbnail 영상을 보여주는 준다. 이때 마우스 왼쪽 버튼으로 클릭하게 되면 우측에 마우스로 클릭된 영상에 대한 pre view 이미지를 보여준다.

Invert Sel.

영상 목록에서 영상의 선택을 반전한다. 즉, 선택한 영상은 선택을 취소하고, 선택하지 않은 영상은 선택한다.

Invert. Sel이 체크된 상태에서 환자 목록, Study 목록, Series 목록을 선택하면, 기본적으로 영상은 선택되지 않은 상태(노란색박스 표시가 없는 상태)로 표시된다.

영상을 Open 했는데 정상적으로 이루어지지 않을 경우, Invert. Sel 항목을 확인하여 원하는 영상에 노란색박스 표시가 있는지 확인한다.

OPEN : 선택된 영상들을 오픈한다.

EXPAND : Thumbnail 영상의 숫자가 많아서 아래의 thumbnail 한줄로 모든 영상들을 확인할 수 없을 때 EXPAND 눌러서 아래 그림과 같이 전체를 볼 수 있게 해 준다.

아래의 EDIT 버튼은 아래의 그림처럼 검사정보를 수정하기 위하여 사용된다. 이때 누가 수정한 것인지 확인하기 위하여 계정확인을 실시한다.

ACCEPT 기능은 Reject 된 이미지를 다시 Accept 하고자 할 때 사용하게 된다. Reject 된 이미지를 선택한 후, ACCEPT 버튼이 active가 되면 클릭하여 accept 할 수 있다.

Image 순서 바꾸기 : 촬영된 영상의 순서를 바꾸고자 할 때 사용된다. 사용 방법은 움직고자 하는 영상을 마우스로 클릭한 채 옮기고자 하는 영상의 앞쪽에 놓고 클릭을 해제한다(drag & drop)

메세지를 확인한 후 클릭한다.

영상 순서 바꾸기 기능은 동일한 Series 내에서만 가능하기 때문에 항상 사전에 한 개의 series 를 선택한 후에 선택된series시리즈내에서 영상의 순서를 변경해야 한다.

Expand를 하지 않은 상태에서의 thumbnail 창에서도 가능하다.

File Export

선택한 영상을 일반 그림 파일로 저장한다. BMP, JPG, PNG, DICOM 등으로 저장할 수 있다. (Config 에서 설정한 환자정보가 overlay 된다.)

- File name은 PatId_StuId_filename으로 저장 된다.

- 저장 시 동일한 이름의 파일이 있으면 File name 뒤에 _1, _2, … 순서로 번호가 붙여 진다.

- Folder name칸에 내용이 있으면 현재 경로에 입력한 Folder name 폴더를 만들고 만들어진 폴더 안에 저장된다.

CD Backup

선택한 영상을 CD매체로 백업한다. 이때 영상데이타를 백업함과 동시에 백업된 영상을 보기위한 viewer 소프트웨어도 같이 저장되게 된다. 이렇게 저장된 CD를 위부로 보내기 위한 도구로도 활용할 수 있다.

CD만 넣게 되면 자동 실행되어서 Viewer에서 영상을 확인할 수 있다.

Backup

선택한 영상을 지정된 폴더로 백업할 수 있다.

이때에도 CDWrite backup 과 같은 CD Publisher와 같은 Viewing software를 같이 저장할지 선택하여 준다. 같이 백업 또는 외부로 내보내고자 할때에도 이 기능을 사용한다.

공 CD를 CDROM 드라이브에 삽입한 다음

백업경로를 CDROM 드라이브로 선택한다.

Backup start를 하여 백업을 시킨다.

아래와 같은 메시지가 뜨면 Yes를 눌러서 최종 마무리 한다.

Restore

백업된 영상을 복원한다.

이 복원기능은 DICOM import의 역할도 할 수 있다. 다른 곳으로부터 받아온 DICOM 파일을 본 DR-console 소프트웨어에 등록하고자 할 때에도 이 기능을 사용할 수 있다. 단, 최종영상이 image processing 되기 전의 영상으로 인식한다.

Image Processing Parameter 설정

영상 처리에 사용하는 인자들이 저장되어 있는 parameter 파일을 지정한다. P/S/M/L는 환자의 조건에 따라 사전에 지정된 PAR파일을 선택할 수 있다. 만약 사전에 설정되어 있지 않다면 각 해당 경로가 빈칸으로 디스플레이 된다.

영상 처리를 하려면 꼭 지정되어야만 한다. 만약 사전에 설정된 경로에 파일이 존재하지 않으면 아래의 process등의 실행버튼들이 모두 disable 된다.

2. 영상에서 측정하려고 하는 눈금자 또는 실제길이를 알고는 물체의 길이만큼 line을 그린다.

3. 길이 측정을 완료하면 아래와 같은 화면이 나타난다.

4. Measure는 영상에서 측정한 길이를 표시합니다.

5. Real에 실제 길이를 mm 단위로 입력하면,

6. Stand (선택한 Bucky)에 계산된 Pixel Spacing값이 표시됩니다.

7. Reset은 계산된 Pixel Spacing값을 무시하고, Aceview.ini 파일에 저장된 Pixel Spacing값(설치 시점의 초기값)을 불러와서 표시합니다.

8. Save는 현재 Pixel Spacing값을 저장하고 화면을 종료합니다.

9. Cancel은 현재 작업을 취소하고 화면을 종료합니다.

Option software 및 function

Viewsub

Option software 인 sub view는 main console software와 동기화 되어 움직이는 소프트웨어이다. 이 소프트웨어는 main console에서 뿐 아니라, 별도의 터치모니터등을 사용하여 제너레이터를 제어하고, 촬영 후 영상의 preview 이미지를 확인확인하고자 할 때 사용할 수 있다.

별도의 옵션 항목이며, 제너레이터가 연동이 되어 있어야만 제대로 할용 할 수 있다.

Exam list에 있는 항목들은 main console에 있는 exam list와 동일하게 같이 움직이며, viewsub 창에서 검사 순서를 바꾸어서 선택하여도 된다.

획득된 영상은 위와 같이 뜨게 되고, 우측의 save 또는 cancel 눌러서 영상에 대한 확인 작업을 해주어야 한다. 만약 여기서 save 또는 cancel를 누르지 않으면 다음 순서로 넘어 갈 수 없다. 단, add-exposure, re-exposure는 자동저장으로 되어 있기 때문에 영상만 잠시 확인 시켜 주고 다음단계로 자동으로 넘어간다.

 feelVIEW

feelVIEW는 main console 모니터외에 별도의 의료용 이나 고해상도 모니터를 사용하여 영상을 좀더 자세히 보고자 할 때 사용하는 소프트웨어이다. 이때에 하나의 workstaion내에 모니터가 연결되어야 하는 전제조건이다.

Option 소프트웨어 이므로 별도로 구매하여야 한다.

기능적으로 main console viewer의 review 모드에서 thumbnail 을 click할때 마다 main console의 영상이 바뀌면서 동시에 feelVIEW의 영상도 바뀌게 된다.
