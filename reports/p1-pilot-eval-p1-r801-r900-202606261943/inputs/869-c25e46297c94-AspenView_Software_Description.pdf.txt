1. Software Description
1.1 Device Description
The AspenView software is designed for use by radiologists and radiology technicians for
the acquisition of digital x-ray images. It interfaces with 3rd party digital x-ray detectors
manufacturer supplied software for the acquisition and storage of digital x-ray images. The
AspenView software then provides a user interface for the viewing, annotating, and other
workstation functions. AspenView includes the ability to receive patient information and
send x-ray images to remote destinations using the DICOM 3.0 protocol.
AspenView software consists of some program module that is developed by ourselves such
as Worklist, Decision Board(Exam), Browser (Worklist), Decision Board(Viewer), Database and
DICOM module and that is developed by some manufacture such as the detector, the
generator and etc.
So the AspenView software is validated at our program module mentioned above only. If
someone wants to be validated manufacture’s s/w, please refer to another document
provided by manufacturer.

1.3 Security Architecture Design
1) CreateAesService Module
The AES key and IV values are generated when AspenView is first executed and are
stored in the WorkList.db. The key and IV are generated using .NET's
System.Security.Cryptography.Aes and
System.Security.Cryptography.RandomNumberGenerator
2) EncryptDicomParameter
This module encrypts DICOM tags. It encrypts the following tags using AES256:
InstitutionName, StudyDescription, ReferringPhysiciansName, RequestingPhysician,
ReasonForTheRequestedProcedure, ScheduledProcedureStepDescription,
InstitutionalDepartmentName, PatientsName, PatientID, PatientsBirthDate,
PatientsSex, PatientsAge, PatientsSize, PatientsWeight, OperatorsName, and
ResponsiblePerson. The encryption is performed using .NET's
System.Security.Cryptography.Aes class.Detector
3) Decrypt DCM File
This module decrypts the tags contained in a DCM file. It first opens the DCM file
using the fo-dicom module, decrypts each tag, and then saves the result.
4) CommonUtils.GetCryptedString
This function is used for password encryption. It returns a hash value of the input
text, utilizing .NET's System.Security.Cryptography.
5) DecryptData, EncryptData

1.6 Intended use (Medical Indication)
AspenView is indicated for use in general radiographic images of human anatomy. It is
intended to replace radiographic film/screen systems in all general-purpose diagnostic
procedures (excluding fluoroscopic, angiographic, and mammographic applications). The
main features of this software are controlling and interfacing the detector, controlling the x-
ray generator acquisition settings, storing acquired images, data management, image
processing and image stitching.

1.11 Operating principle (how to achieve the intended
purpose)
A radiograph is an X-ray image obtained by placing a part of the patient in front of an X-
ray detector and then illuminating it with a short X-ray pulse. To take an X-ray image, the
operator sets the patient in front of the detector and sets the parameter values of the high
voltage generator to control the intensity of X-ray for each body part. The detector changes
the X-rays coming through the part of the patient to digital signals and makes digital
images.
An Acquisition program is to acquire, process, storage and view the taken X-ray image by
detector or Image plate. After exposure and readout, the raw imaging data must be sent
and processed for display on the computer. Image processing is one of the key features of
digital radiography, greatly influencing the way the image appears to the radiologist.
Image processing is used to improve image quality by reducing noise, removing technical
artifacts, and optimizing contrast for viewing.
