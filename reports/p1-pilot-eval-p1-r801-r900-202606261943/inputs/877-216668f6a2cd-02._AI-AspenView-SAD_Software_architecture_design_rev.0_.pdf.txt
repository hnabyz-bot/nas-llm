3. Security architecture views
4. Security Modules Specification
4.1 CreateAesService Module
The AES key and IV values are generated when AspenView is first executed and are stored in
the WorkList.db. The key and IV are generated using .NET's System.Security.Cryptography.Aes
and System.Security.Cryptography.RandomNumberGenerator.
4.2 EncryptDicomParameter
This module encrypts DICOM tags. It encrypts the following tags using AES256:
InstitutionName, StudyDescription, ReferringPhysiciansName, RequestingPhysician,
ReasonForTheRequestedProcedure, ScheduledProcedureStepDescription,
InstitutionalDepartmentName, PatientsName, PatientID, PatientsBirthDate, PatientsSex,
PatientsAge, PatientsSize, PatientsWeight, OperatorsName, and ResponsiblePerson. The
encryption is performed using .NET's System.Security.Cryptography.Aes class.

4.3 Decrypt DCM File
This module decrypts the tags contained in a DCM file. It first opens the DCM file using the fo-
dicom module, decrypts each tag, and then saves the result.
4.4 CommonUtils.GetCryptedString
This function is used for password encryption. It returns a hash value of the input text,
utilizing .NET's System.Security.Cryptography.
4.5 DecryptData, EncryptData
Using the AES key and IV values, an Encryptor and Decryptor are created. The encryption and
decryption processes are performed using .NET's System.IO.MemoryStream,
System.Security.Cryptography.CryptoStream, and
System.Security.Cryptography.CryptoStreamMode.
4.6 NotiBar
Screen saver function setting and unused session termination. The user will be automatically
logged out if no mouse or keyboard events occur for a certain period of time (10 minutes).
