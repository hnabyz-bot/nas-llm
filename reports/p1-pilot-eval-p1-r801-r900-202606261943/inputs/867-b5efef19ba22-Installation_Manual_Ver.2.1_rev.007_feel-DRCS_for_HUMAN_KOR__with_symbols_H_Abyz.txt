Introduction

feel-DRCS 는 imfoU 사의 Digital Radiography Control Software의 제품명 입니다.

feel-DRCS는 flat-panel 또는 CCD 등의 DR detector로부터 영상 획득 및 임상적용을 위한 빠른 속도의 영상처리, 그리고 PACS 연동을 위한 DICOM 서비스를 제공하는 DR console 또는 DR control software 입니다.

이 매뉴얼은 feel-DRCS 소프트웨어를 installation 및 operation 하기 위한 내용을 정리되어 있습니다.

IMFOU CO.,LTD.

#1105, Daerung PostTower7, 48, Digital-ro 33-gil, Guro-gu, Seoul, 08377. Korea.

Purpose

feel-DRCS 는 FS-MLW(Faster Specialized Multi Layered Wavelet) 기술을 기반하여, 각종 detector나 X-Ray Generator와 integrated 된 Operation Console System 소프트웨어입니다.

feel-DRCS 는 절대적인 사용자 입장에서 개발된 제품으로, 다년간 임상사이트인 병원에서 디자인 되었으며, 서비스를 위한 DR 제조사들의 서비스 엔지니어들에 의해 완성된 제품입니다. 그러므로 쉽고, 간결하면서도 다양한 기능들이 포함된 최상의 User interface로 개발된 DR Console system입니다.

feel-DRCS 는 feel-TK를 기반으로 내장되어 있기 때문에 보다 빠른 속도로 Noise와 각종 artifact 제거되고, 보다 선명한 생체조직을 볼 수 있습니다. 그리고 독자적인 calibration 기능이 내장되어, 모두 종류의 detector에 대한 기본 calibration data(offset image and gain image)을 설정할 수 있습니다. 그리고 defect view를 제공하여 진행성 defect들을 처리할 수 있습니다.

feel-DRCS 는 전세계적으로 판매되고 있는 각종 detector들에 대해서 이미 소프트웨어 검증 및 판매가 되어서 임상필드에서 사용되고 있습니다. DR제조사가 사용하고자 하는 detector를 손쉽게 설정에서 타입만 변경하면 바로 콘솔시스템의 소프트웨어로 사용이 가능합니다. 그리고 통합 DR 시스템인 X-ray 의 generator operation panel을 소프트웨어 내부에 integration 되어 있어, APR, AEC, Ready/Exposure등을 제어가 손쉽게 가능합니다.

feel-DRCS 는 각종 X-ray Generator와 연동이 되어, APR 기능 등을 지원하여, 하나의 콘솔 소프트웨어에서 기존의 Operation Panel의 역할을 같이 수행할 수 있게 하여 분리된 DR제품이 아닌 하나의 통합된 DR제품을 지원합니다.

feel-DRCS 는 단순히 소프트웨어를 공급하는 것이 아니라, 다년간 경험했던 노하우를 DR 제조사에게 전달해 드리며, 동시에 컨설팅까지 같이 지원해 줌으로써 조력자(implement for you)의 역할을 다 할 것입니다.

2-1. 사용용도

Feel-DRCS는 방사선사 및 의사들이 일반 X선 영상을 획득하기 위한 용도로 사용된다.그래서 병원에서 근무하는 의사 및 방사선사외에는 사용할 수 없다. 또한 이 소프트웨어는 서비스 엔지니어에 의해서만 설치되고 방사선기사 및 의사에 의해서만 사용되어진다.

현재 PCI/LPCI/LPCIe-7250 과 USB-7250 모델이 지원된다.

ii. Ready Delay

Software 적으로 Prep 신호 입력 후 Exposure 신호가 입력이 되었을 경우 Generator가 준비가 되지 않은 상태에서 X-ray가 조사되는 것을 방지하기 위해 Ready Delay를 설정한다.

ETC

i. 12 bit send

전송 시에 12비트 DICOM 파일로 전송하고자 하는 경우에 선택합니다. 원본 데이터 값이 14비트 이상임으로 이 선택옵션은 PACS 가 12비트까지 처리가 되는 경우에 선택하고, 나머지는 그냥 체크해제상태로 둡니다.

ii. Interpolation

Review 모드로 진입 시에 영상을 로드하게 됩니다. 이때에 모든 이미지 로드 시에 interpolation 해서 로드하고자 하는 경우에 사용합니다. 단 이 기능을 사용하는 경우에는 로드할 때 많은 시간이 소요됨으로 체크해제상태로 두는 것을 권합니다.

iii. AutoCrop

 이 기능은 촬영 시에만 동작하며, 촬영시 영상 획득 및 프로세싱 후에 collimator 을 이용한 촬영된 영상에 대하여 자동으로 collimator 된 영역을 검출하여 crop 해 주는 기능입니다. 이 기능은 촬영시에 단 한번만 자동으로 동작 되기 때문에 crop 영역을 해제하면 다시 수동으로 작업하셔야 합니다.

iv. Virtual Keyboard

 Touch 모니터를 사용하는 경우에 문자입력을 모니터상에서 터치를 통하여 입력할 경우에 사용합니다. 이 경우에 windowing 조절 시에 터치를 이용하여 사용할 수 있습니다. 문자를 입력할 수 있는 모든 TEXT 박스를 선택하면 자동으로 화면에 keyboard가 open하게 됩니다.

v. Reject

 촬영 후에 획득한 영상에 대하여 reject 하는 기능이다. Reject는 reject 마크를 표시하고 사용할 수 없도록 할뿐 데이터 자체가 지워지지 않는다. 만약 이 기능이 체크해제가 되어 있는 경우에는 cancel이 되면서 삭제가 됩니다.

vi. Analysis

 체크가 된 이후로 촬영된 모든 영상에 대하여 환자수, 획득한 영상 개수, DICOM sending 및 reject 된 이미지수 등을 분석할 수 있는 기능이다.

vii. Stitching Processed Image

 stitching 시에 이미 처리된 이미지들로 처리하고자 하는 경우에 체크합니다. 만약 스티칭 후에 원본이미지에서 다시 영상처리를 하고자 하는 경우에는 이 기능 체크해제 합니다

viii. Sensitivity

 Exposure Index를 표시하고자 할 경우에 해당 디텍터의 감도를 입력합니다.

ix. Use Ralco

 Ralco Auto collimator와 통신하고자 하는 경우에 체크하여 사용합니다. 세부적인 설치 방법은 별도의 설치 매뉴얼을 참조하시기 바랍니다.

x. Show Wifi UI

 Wifi Detector를 사용할 경우, Wifi 상태 및 Battery 상태 표시

xi. No Use Predefine shutter

 Pre-define 된 shutter의 on/off 설정

ViewSub

ViewSub 모니터는 Generator와 연동이 가능한 모델인 경우에 사용합니다. Main console 모니터외에 별도의 모니터가 장착되어 사용하는 경우입니다. 만약 하나의 모니터를 사용하는 경우에는 이 해상도는 자동으로 인식되어 사용되니 체크할 필요가 없습니다. 하나의 PC에 2개의 모니터를 사용하는 것으로 Main console은 항상 1번 위치의 모니터를 사용합니다. 그리고 viewsub는 2번 모니터를 사용합니다. Pos X 와 Pos Y는 2번 모니터의 시작점 위치입니다. 이 위치는 Window 시스템의 display 정보를 확인하여 입력합니다.

DICOM MWL interface by Text file

Principles of interfacing

Define order argument

Method of Interfacing

(1) Filename

- Order_Date.mwl (ex. Order_20140101123456.mwl)

(2) Store destination folder

- C:\feel-DRCS\OCS\

(3) Argument delimiter

- “,” : You can use this character in all of Text file.

(4) Study delimiter

 - CR : Carriage return (You must input CR the final patient item.)

(5) Unknown value

- Blank

(6) Mandatory Argument

 - Patient ID/Name/Sex/Modality/Date of Study Scheduled
