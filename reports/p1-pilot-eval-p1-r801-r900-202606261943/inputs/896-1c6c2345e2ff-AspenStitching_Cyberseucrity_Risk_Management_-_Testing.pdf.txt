2. Vulnerability Scanning Test Result
• Date of Test: 10/10/24
• Test Objective: Identify potential vulnerabilities in third-party libraries, runtime libraries,
and compilers used within the system.
2.1 Scope
• Include the specified libraries and components only.
• Communication protocols used for image retrieval
• Database for storing software configurations and credentials.
2.2 Test Methodology
2.2.1 Reconnaissance
• Reviewed software documentation and system architecture diagrams to understand the
components and communication pathways involved in the AspenStitching software.
• Analyzed transmission traffic between the stitching software and the imaging detector to
identify potential attack surfaces and communication protocols.
2.2.2 Vulnerability Scanning
• Utilized static and dynamic analysis tools to scan the X-Ray Image stitching software
binaries for known vulnerabilities and common security weaknesses.
• Conducted port scanning and network vulnerability assessments to identify potential
entry points and security flaws in network configurations.
2.2.3 Manual Testing
• Manually inspected the stitching software codebase for security vulnerabilities, including
buffer overflows, input validation flaws, and insecure data storage practices.
• Conducted user interface testing to identify potential privilege escalation vulnerabilities
and unauthorized access paths within the application.

2.3 Vulnerability Findings
2.3.1 Medium-Risk Vulnerabilities
• Weak input validation in user-controlled parameters, leading to potential SQL injection
and command injection vulnerabilities in database queries and system commands.
2.3.2 Low-Risk Vulnerabilities
• Lack of proper session management controls in the stitching software, increasing the
risk of session hijacking and unauthorized access to sensitive patient data.
• Information disclosure vulnerability detected in error messages, leaking sensitive
system information that could aid attackers in further exploitation.
• SSL/TLS configuration issues identified, including the use of weak cipher suites and
outdated SSL protocols, potentially exposing the system to man-in-the-middle attacks.
2.4 Mitigation Measures
2.4.1 Remediating Medium-Risk Vulnerabilities
• SQL Injection Vulnerability:
o Implement stronger input validation and parameterized queries to sanitize user
inputs and mitigate the risk of injection attacks.
o Conduct code review and security testing to identify and remediate similar
vulnerabilities in other parts of the application.
2.4.2 Remediating Low-Risk Vulnerabilities
• Authentication Bypass Vulnerability:
o Implement proper authentication mechanisms such as strong password policies
and multi-factor authentication to prevent unauthorized access.
o Apply security patches or updates to address the authentication bypass
vulnerability.
o Implement session expiration after extended period of time.
• Weak Password Policy:

3. Penetration Test Report
• Date of Test: 10/10/24
• Test Objective: To simulate real-world attacks and attempt to compromise the security
of the AspenStitching software, identifying potential entry points and vulnerabilities.
3.1 Scope
• AspenStitching Software
• Communication protocols used for image transmission between the stitching software
and the imaging detector.
• Network infrastructure supporting the stitching software deployment.
3.2 Test Methodology
3.2.1 Reconnaissance
• Gathered information about the target environment (AspenStitching), including system
architecture, network topology, and software components.
• Identified potential attack surfaces and entry points through open-source intelligence
(OSINT) and network reconnaissance techniques.
3.2.2 Vulnerability Scanning
• Utilized automated vulnerability scanning tools Nessus and OpenVAS to identify known
vulnerabilities in the AspenStitching software and associated systems.
• Conducted port scanning and service enumeration to identify open ports and running
services for further exploitation.
3.2.3 Exploitation
• Attempted to exploit identified vulnerabilities to gain unauthorized access to
AspenStitching and backend systems.

• Utilized penetration testing frameworks and custom scripts to simulate real-world attack

3.2.4 Post Exploitation
• Performed privilege escalation techniques to escalate access rights and gain

• Explored the target environment to identify additional systems and sensitive data

3.3 Test Scenarios
3.3.1 SQL Injection Attack
• Attempted to exploit SQL injection vulnerabilities in the stitching software's search
functionality to extract sensitive patient data from the backend database.
3.3.2 Remote Code Execution
• Attempted to exploit buffer overflow vulnerabilities in the image parsing module to
execute arbitrary code on the user's system and gain remote access to AspenStitching
software.
3.3.3 Man-in-the-Middle Attack
• Attempted to intercept and manipulate network traffic between the stitching software
and the imaging detector to modify transmitted X-ray images or inject malicious content.
3.3.3 Network Crawling
• Attempted to intercept and exploit open ports that can be used to gain access into the
software and inject malicious content.

3.5 Mitigation Measures
3.5.1 Improved Encryption of Communication
• Implement secure communication protocols such as TLS/SSL to encrypt data
transmission between the stitching software and the imaging detector.
• Enforce security updates of operating system to raise security standard of the operating
hardware
3.5.2 Improved Defense Against for SQL Injection:
• Patch the search inputs and all searchable fields allowing inputs of the application to
enforce stricter validation and better error handling.
3.5.3 Improved Authentication Policy:
• Implement proper authentication mechanisms such as stronger password policies and
multi-factor authentication to prevent unauthorized access.
• Apply security patches or updates to address the authentication bypass vulnerability.
