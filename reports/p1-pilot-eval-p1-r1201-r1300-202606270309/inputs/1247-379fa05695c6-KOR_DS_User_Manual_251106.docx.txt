디텍터 연결 방식

사용자는 G1717CW, G1717GCW 디텍터와 라우터 사이를 유선 또는 무선 연결 방식으로 구성할 수 있으며, 필요할 경우 사용 환경에 적합한 연결 방식을 선택하여 사용할 수 있습니다.

유선 연결 모드

LAN 케이블을 통해 PC와 직접 연결됩니다. G1717CW, G1717GCW 디텍터는 제너레이터와 연결될 수 없습니다.

Soft AP 모드

AP 모드는 별도의 AP 없이 디텍터를 AP로 사용하여 PC와 직접 무선 연결할 수 있습니다.

# 디텍터 & PC (워크스테이션)

PC와 G1717CW 600 / G1717GCW 600 / G1717GCW 1000 디텍터는 무선 방식으로 연결됩니다. 이 경우에, 디텍터는 AP로서 작동하며 PC는 스테이션으로서 작동합니다.

무선 연결에서 디텍터는 기본적으로 배터리에서 전원을 공급받습니다. 단 AC/DC 어댑터를 연결하면, 디텍터는 어댑터로부터 전원을 공급받습니다.

무선 연결 모드

G1717CW, G1717GCW 디텍터는 별도의 라우터를 통해 PC (워크스테이션)과 연결되며, PC에 영상 및 데이터를 전송합니다.

소프트웨어 무선 연결

본 디텍터는 3가지 연결 방식을 제공합니다.

Wired : 유선 연결입니다.

Wireless : 무선 연결입니다. 디텍터는 지정된 SSID를 가지는 라우터를 검색하고 연결합니다.

AP : 무선 연결입니다. 디텍터는 지정된 SSID, Channel, 보안 방식 및 Password 설정을 사용하여 AP를 제공하며, 무선 통신을 제공하는 PC는 이 AP에 연결을 시도할 수 있습니다.

연결 모드

소프트웨어의 상단에 위치한 Connection Config 버튼을 눌러 다음 화면에 진입합니다.

Set 또는 Get 버튼을 눌러 디텍터의 Connection Mode를 설정하거나 조회할 수 있습니다. 디텍터는 5가지의 Connection Mode를 제공하며 각 모드는 아래와 같은 방식으로 동작합니다.

Ethernet Only : Wired(유선)모드입니다. Thether Cable을 통해서만 통신할 수 있습니다.

Allow Wireless : Wireless 허용 모드입니다. 기본적으로 Thether Cable을 통해서 통신할 수 있으며, Cable이 물리적으로 분리되는 경우 자동으로 Wireless 연결을 시도합니다. (단, Wireless 연결 정보가 유효해야 합니다.)

Allow AP : AP 허용 모드입니다. Thether Cable을 통해서 통신할 수 있으며, Cable이 물리적으로 분리되는 경우 자동으로 Host AP를 시도합니다. (단, AP 설정값이 유효해야 합니다.)

Wireless Only : Wireless 모드입니다. 디텍터의 전원이 ON된 시점부터 상시로 Wireless 상태를 유지합니다. 설정된 Wireless 연결 정보를 사용하여 연결에 성공할 때까지 라우터에 연결을 지속적으로 시도합니다. Thether Cable이 부착되더라도 Wired 모드로 전환되지 않습니다. (만약 설정된 Wireless 연결 정보가 유효하지 않다면 디텍터는 통신이 불가능한 상태가 됩니다. 이러한 경우 디텍터의 AP 버튼을 5초 이상 길게 눌러 Connection Mode을 Allow AP로 초기화한 뒤 Thether Cable 또는 AP를 사용하여 설정값을 확인하십시오.)

AP Only : AP 모드입니다. 디텍터의 전원이 ON된 시점부터 상시로 AP 상태를 유지합니다. 설정된 AP 정보를 사용하여 Host AP를 수행합니다. 사용자는 디텍터의 SSID를 무선 네트워크 목록에서 찾아 연결을 시도할 수 있습니다. Thether Cable이 부착되더라도 Wired 모드로 전환되지 않습니다. (만약 설정된 AP 정보가 유효하지 않거나 Host AP에 실패한 경우 디텍터는 통신 불가능한 상태가 될 수 있습니다. 이러한 경우 디텍터의 AP 버튼을 5초 이상 길게 눌러 Connection Mode를 Allow AP로 초기화한 뒤 Thether Cable을 사용하여 설정값을 확인하십시오.)

IP

디텍터는 모든 Connection Mode에 대해서 하나의 IP를 사용하며 이 페이지에서 IP를 변경할 수 있습니다. IP의 초기값은 192.168.0.54입니다.

디텍터는 255.255.255.0의 서브넷 마스크를 사용하기 때문에 192.168.0은 고정값으로 두고 마지막 옥텟만을 변경하는 것을 권장합니다. (단, 마지막 자리에 0, 1 및 255는 사용할 수 없습니다.)

디텍터의 IP가 정상적으로 변경되었다면 현재 연결은 즉시 끊어지며 디텍터는 새로운 IP로의 통신을 개시하게 됩니다.

다음으로 Bright Files 목록 하단의 Add Dark 버튼으로 Dark 파일을 선택합니다.

Bright 파일과 Dark 파일을 추가하면 다음과 같이 나타납니다.

우측의 Calibration Files 항목에서 Create 버튼을 누르면 Calibration에 필요한 자료가 생성됩니다.

Bright 파일은 각 파일의 median 값에 따라 Calset 파일을 생성합니다. 이 때, 하단의 Tolerance

설정에 따라 Calset 파일을 개수가 달라집니다.

예) Tolerance = 100, Bright_1 Median = 1000, Bright_2 Median = 1050🡪 Bright_1과 Bright_2의 Median 차이가 100 이하이므로 하나의 Calset 파일이 생성됨Tolerance = 100, Bright_1 Median = 1000, Bright_2 Median = 1200🡪 Bright_1과 Bright_2의 Median 차이가 100 이상이므로 두개의 Calset 파일이 생성됨

세척

오염된 부분은 70% IPA (Isopropyl-alcohol)으로 세척합니다.

세척 전, 디텍터의 전원을 끄고 배터리를 제거합니다.

IPA 혹은 다른 세척용 액체와의 직접적인 접촉을 방지하기 위해 방수 장갑을 착용한 상태로 세척을 진행합니다.

디텍터에 직접적으로 IPA를 뿌리거나 붓지 않습니다. 면이나 부드러운 천에 IPA 용액을 적셔 닦아냅니다.

IPA 혹은 다른 세척용 액체가 디텍터 안에 들어가지 않도록 주의합니다.

세척 후, IPA가 완전히 마를 때까지 기다립니다.

전자파(EMC) 적합성 및 지침

규제 준수

이 장비는 의료기기에 대한 IEC 60601-1-2 EMC 표준을 준수합니다.

이 장비는 EMC에 대한 특별한 예방 조치가 필요하며 사용 설명서에 제공된 EMC 정보에 따라 설치 및 가동해야 합니다.

시스템 근처에서 휴대전화를 포함한 무선 장치를 사용하지 마십시오. 모든 무선 장치는 EMC 표준 준수 여부와 상관없이 전자기 간섭을 야기할 수 있으며, 근처에서 사용할 경우 시스템이 오작동할 수 있습니다.

본 장비를 다른 장비 옆에 두거나, 쌓은 채로 사용하지 마십시오. 만약 다른 장비 옆이나 쌓아서 사용해야 하는 경우, 해당 장비를 테스트하고 검증하여 해당 사용 구성에서 정상적으로 작동하는지 확인해야 합니다.

규격 준수에 대한 책임자가 명시적으로 승인하지 않은 변경 또는 수정은 장비를 작동할 수 있는 사용자의 권한을 무효화할 수 있습니다.

주변 장치에 대한 모든 상호 연결 케이블은 기술적으로 금지된 경우를 제외하고 차폐되고 적절히 접지되어야 합니다. 적절하게 차폐 및 접지되지 않은 케이블을 사용하면 장비가 무선 주파수 간섭을 일으킬 수 있습니다.

지정된 케이블 또는 액세서리가 아닌 다른 케이블을 사용하면 전자파 방출이 증가하거나, 전자파 내성이 감소할 수 있습니다. 다른 장비에서 간섭(EMC)이 감지되는 경우, 다른 장비를 이 장비에서 멀리 두십시오.

이식형 심장 박동 조절기 또는 이식형 심장 박동 조절기를 사용하는 환자에게 이 시스템을 사용하는 경우, 환자에게 펄스 형태의 지속적인 X선 노출로 인해 심장 박동 조절기 또는 심장 박동 조절기가 오작동할 수 있음을 알려야 합니다.

이 시스템을 사용할 때, X선이 환자의 이식형 심장 박동 조절기 또는 이식형 심장 박동 제세동기에 직접 노출되지 않도록 하고 노출 시간을 가능한 한 짧게 유지해야 합니다. 이 장비는 다른 장비가 CISPR EMISSION 요구 사항을 준수하더라도 다른 장비의 간섭을 받을 수 있습니다.
