Overview

Purpose

The purpose of this document is to show a cyber security architecture view for Portable X-ray System(HnX-P1, HnX-PB).

Scope

This document is limited to HnX-P1 and HnX-PB, the Portable X-ray System.

In addition, three of the four architectural views corresponding to the FDA requirements are Global system view, Updateability/Patchability view, and Security use case view. Multi patient harm view is excluded because it is not our product.

Architecture view

Data Flow Diagram (Global System View)

A1: Configuration - Receive kV, mAs, APR setting value information.

A2: Calibration - kV, mAs, filament current (pre-heat) compensation value settings.

A3: Data acquisition - Digital & analog data collection.

A4: APR - Saving kV and mAs in 10 channels

A5: X-ray control - Control power on/off for X-ray output, error status output on/off.

A6: Battery charging - Battery charge current/voltage regulation and monitoring.

A7: Battery monitoring - Battery charging status monitoring

A8: Error checking - Checking for occurrence of low voltage, overvoltage, low current, overcurrent, and overheating

A9: Bluetooth - For setting up medical devices and receiving x-ray images via Bluetooth communication

A10: Display - Setting value setting status, X-ray image display.

A11: SID - Distance measurement.

A12: Hardware Interfaces - Physical ports, including USB type C(For power charging and receiving firmware update data.) and RJ-11(For connecting a hand switch).

A13: Open source 1 – STMicroelectronics Open-Source Bundle

A14: Open source 2 - KISA Crypto Module

Multi-Patient Harm View

This product does not correspond to Multi-Patient Harm because it only matches 1:1 with the specified console software via Bluetooth.

Updateability and Patchability View

Firmware update is possible through the USB C type port. (However, direct updates in the field are not possible, and the product must be returned to the manufacturer or service center for updates.)

The port only functions for charging the battery inside the product and for firmware updates, and does not function for any other functions.

Important information cannot be acquired or functional operations cannot be impaired during firmware updates. (Test report no. KTR-QI-Y10053-F10(00), Penetration test PT-1)

The RJ-11 port is a hand switch connection port, and all other unnecessary interfaces are blocked. . (Test report no. KTR-QI-Y10053-F10(00), Penetration test PT-4)

Security Use Case View

Communicates with external products (console software) via Bluetooth function.

Integrity check is used to reject tampered packets and abnormal file transfers. (Test report no. KTR-QI-Y10053-F10(00), Penetration test PT-2, PT-3)
