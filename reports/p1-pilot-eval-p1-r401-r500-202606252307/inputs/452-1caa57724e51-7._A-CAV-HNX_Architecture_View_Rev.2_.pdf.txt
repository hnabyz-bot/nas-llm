1. Overview
1.1 Purpose
The purpose of this document is to show a cyber security architecture view for Portable X-ray
System(HnX-P1, HnX-PB).
1.2 Scope
This document is limited to HnX-P1 and HnX-PB, the Portable X-ray System.
In addition, three of the four architectural views corresponding to the FDA requirements are Global
system view, Updateability/Patchability view, and Security use case view. Multi patient harm view
is excluded because it is not our product.
2. Architecture view
2.1 Data Flow Diagram (Global System View)
• A1: Configuration - Receive kV, mAs, APR setting value information.
• A2: Calibration - kV, mAs, filament current (pre-heat) compensation value settings.
• A3: Data acquisition - Digital & analog data collection.
• A4: APR - Saving kV and mAs in 10 channels
• A5: X-ray control - Control power on/off for X-ray output, error status output on/off.
• A6: Battery charging - Battery charge current/voltage regulation and monitoring.
• A7: Battery monitoring - Battery charging status monitoring
• A8: Error checking - Checking for occurrence of low voltage, overvoltage, low current,

2.2 Multi-Patient Harm View
This product does not correspond to Multi-Patient Harm because it only matches 1:1 with the
specified console software via Bluetooth.
2.3 Updateability and Patchability View
• Firmware update is possible through the USB C type port. (However, direct updates in the
field are not possible, and the product must be returned to the manufacturer or service
center for updates.)
• The port only functions for charging the battery inside the product and for firmware
updates, and does not function for any other functions.
• Important information cannot be acquired or functional operations cannot be impaired
during firmware updates. (Test report no. KTR-QI-Y10053-F10(00), Penetration test PT-
1)
• The RJ-11 port is a hand switch connection port, and all other unnecessary interfaces are
blocked. . (Test report no. KTR-QI-Y10053-F10(00), Penetration test PT-4)
