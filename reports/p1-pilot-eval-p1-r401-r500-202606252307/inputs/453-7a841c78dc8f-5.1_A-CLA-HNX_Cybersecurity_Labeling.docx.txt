This section will outline HnX-R1’s security measures, maintenance, and best practices for the end users to manage cybersecurity risks associated with the modules and the use of the software.

Data flow diagram

< Data Flow Diagram >

CRM-1~CRM-25(A1~A25): These assets are entities implemented in the console application and consist of a database, data manipulation process, detector, and communication interface between PACS/worklist servers and a GUI interface for the console program to implement its functions.

CRM-1~CRM-6(A1~A6): Console Subcomponents, SQL database structure implemented in Console application

CRM-7(A7):User interface with console accessing by console user(keyboards, Display interfacing, IO device etc)

CRM-8~CRM-21(A8~A21): Console Subcomponents, Console Application Process : The console application process is comprised of functions responsible for the image acquisition station, such as post-processing images acquired from the detector, matching them with the worklist, transmitting this information to an independent PACS server within the hospital, and communicating with the detector.

CRM-22(A22): Console – PACS/Worklist server ethernet communication interface

CRM-23 ~CRM-25(A23-25): Console – Detector ethernet communication interface

A26~A32: These assets are entities implemented in the detector inside and consist of a database, data manipulation process, interface for the console program to communication its image data.

CRM-26~CRM-27(A26~27): Detector subcomponents, The detector firmware process is a stand-alone board that acts as the CPU in the detector, manages the detector storage system and communication modules, and configures overall image communication and detector settings.

CRM-28~CRM-29(A28~29): Detector subcomponents, Detector System Storage (store) serves as the memory required to store and communicate images from the detector.

CRM-30~CRM-31(A30~A31) : communication interface between Detector and console implementation by software communication protocol(Socket API)

CRM-32(A32): Detector subcomponents, Communication and data requests are communication protocols for image data and transmission between firmware processes, communication interfaces, and data system storage, and are passages for transmitting images and data.
