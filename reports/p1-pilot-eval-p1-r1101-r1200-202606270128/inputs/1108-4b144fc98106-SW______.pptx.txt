## ppt/slides/slide5.xml
SiMD — Patch 버전 + 참고사항 버전 변경 시나리오 🇰🇷 MFDS 🇺🇸 FDA 🇪🇺 EU MDR Patch UI 오타/레이아웃 수정 내부관리 내부 QMS 문서화 내부 변경관리 성능 무관 버그 수정 (로그 파일 날짜 오류 등) 내부관리 내부 QMS 문서화 내부 변경관리 사이버보안 전용 패치 (기능 변경 없음) 경미변경 (즉시/연차보고) 내부 QMS (510(k) 불요) 내부 변경관리 + PMS 문서화 SiMD 주요 참고사항 • 한국: 운영환경 변경은 가이드라인에서 &apos;업그레이드&apos; 해당. 단, 성능·안전성 영향 없음 입증 시 변경신고 가능 • FDA: OS 변경은 &apos;porting to a new OS&apos; → 아키텍처 변경으로 분류. ## ppt/slides/slide7.xml
SaMD — Patch 버전 + PCCP 안내 버전 변경 시나리오 🇰🇷 MFDS 🇺🇸 FDA 🇪🇺 EU MDR Patch UI 버그 수정 (날짜 표시 오류 등) 내부관리 내부 QMS 내부 변경관리 성능 무관 버그 수정 (로딩속도, 메모리 누수) 내부관리 내부 QMS 내부 변경관리 보안 패치 (인증 취약점, 라이브러리 업데이트) 경미변경 (즉시보고) 내부 QMS (510(k) 불요) 내부 변경관리 + PMS 문서화 PCCP (Predetermined Change Control Plan) — SaMD 핵심 전략 PCCP란? FDA에 사전에 변경 계획을 제출·승인받아, 해당 범위 내 변경은 new 510(k) 없이 LTF로 관리하는 제도 적용 대상: AI/ML SaMD에 특히 유용 (모델 재학습, 파라미터 조정 등을 사전 계획 내에서 관리) 포함 내용: SPS(SaMD Pre-Specifications: 변경 범위 정의) + ACP(Algorithm Change Protocol: 학습/변경 방법론) 실무 팁: 초기 510(k)/De Novo 인허가 시 PCCP를 함께 제출하면 이후 모델 업데이트의 규제 부담을 크게 줄일 수 있음

## ppt/slides/slide8.xml
SiMD vs SaMD — 규제 차이 핵심 요약 구분 기준 SiMD (내장형) SaMD (독립형) SW 변경 시 HW 연동 고려 필수. SW 변경이 HW 성능/안전에 영향 → HW 변경 플로차트도 병행 검토 불필요 (SW 독립적) 단, 배포 플랫폼은 별도 평가 FDA 변경 가이드라인 SW 변경 가이드 + 일반 기기 변경 가이드 (2017) 동시 적용 SW 변경 가이드 주로 적용 AI/ML의 경우 PCCP 가이드 추가 EU MDR 분류 규칙 HW 기기의 등급을 따름 (Rule 9, 10, 11 등 해당 기기 규칙) Rule 11 독립 적용 일반적으로 Class IIa 이상 한국 경미변경 처리 별표3 &apos;내장 소프트웨어&apos; 관련 조항 적용 별표3 &apos;독립형 소프트웨어&apos; 조항 (2022년 네거티브 규제 개선) AI/ML 추가 요구사항 드묾 (대부분 전통적 알고리즘) FDA PCCP, AI/ML Action Plan, GMLP + EU AI Act 연계 아키텍처 변경 민감도 높음 — OS 변경이 기기 전체 안전성(전기·기계적)에 직접 영향 중간 — 클라우드/앱 배포 유연 단, 성능 검증은 동일하게 요구

YES → 거의 확실히 신규 심사 대상 NO → Step 3으로 이동 Step 3 안전성/유효성에 영향 가능? 새 hazardous situation 도입 여부? 임상 기능/성능 규격 변화 가능? YES → 규제기관별 플로차트 상세 평가 NO → Step 4로 이동 Step 4 V&V 결과 확인 V&V 수행 시 예상치 못한 이슈 발견? 이슈 발견 → Step 3으로 돌아가 재평가 이슈 없음 → 내부 문서화 (LTF/경미변경) 누적 변경(Cumulative Changes) 주의: 개별 Patch/Minor가 누적되면 원래 인증 상태와 달라질 수 있음. ## ppt/slides/slide10.xml
실무 팁 & 참조 규격 버전 정책 사전 수립 SOP에 Major/Minor/Patch 정의서 포함. 규제기관은 번호보다 변경 내용·위험 평가 근거를 봄 위험 평가 선행 모든 변경에 Risk Assessment 먼저 수행. FDA 플로차트 핵심 질문: &apos;could this significantly affect safety or effectiveness?&apos; LTF/경미변경 보고서 충실화 &apos;심사 불요&apos; 결론도 판단 근거 문서화 필수. Inspection 시 방어 가능해야 함 AI/ML SaMD는 PCCP 활용 초기 인허가 시 PCCP 함께 제출. 이후 모델 업데이트 시 규제 부담 대폭 감소 주요 참조 문서 🇰🇷 의료기기 허가·신고·심사 등에 관한 규정 (별표3) / 소프트웨어 허가·심사 가이드라인 (2023.7) 🇺🇸 FDA: Deciding When to Submit a 510(k) for a Software Change (2017) / PCCP Guidance (2023~2024) 🇪🇺 MDCG 2020-3 rev.1 (Significant Changes) / MDCG 2019-11 rev.1 (SW Classification, 2025) 📋 IEC 62304 (SW Lifecycle) / ISO 14971 (Risk Management) 본 가이드는 참고 목적이며, 최종 판단은 각 규제기관 가이드라인 원문 및 사전 상담(Pre-Submission 등)을 통해 확인하시기 바랍니다.
