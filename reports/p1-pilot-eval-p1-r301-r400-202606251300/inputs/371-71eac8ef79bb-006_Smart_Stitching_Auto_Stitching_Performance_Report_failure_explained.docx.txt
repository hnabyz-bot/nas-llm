Test Condition

OS : Microsoft Windows 10 Pro(64bit)

CPU : Intel(R) Core(™) i5-6500 CPU @3.20GHz 3.19GHz

RAM : 8GB

Graphic Card : NVIDIA Geforce GT 730

HDD : 1TB

Test Methods

Download the DICOM image to a local storage. Import the locally stored image and perform the test.

The number of stitching images should be 2~5.

Test acceptance criteria

Compared with manual stitching, the error range for auto stitching should be less than 15 pixels each in the horizontal and vertical direction.

Obtain manual stitching images from radiologist(Kim Joo-Young).

For Spine image capture, the number of vertebrae (C-Spine: 7, T-Spine: 12, L-Spine: 5) should be correct. - [See the Figure 2]

Get confirmation from the radiologist

If there is a tape measure, it should be evenly aligned. [See the Figure 3]

Get confirmation from the radiologist.

If any one of 1~3 conditions is not met, it is regarded as failure. [See the figure 4]

Test result

Based on 2 input images, the auto stitching time is about 1 second.

The cause of the failure in image stitching has been analyzed to be patient movement during the acquisition of x-ray images. See [Table 1] No.1, No. 3

The auto stitching result is greatly affected by the auto stitching parameter and the optimized parameter value may be different depending on the location of the redundant area of the image. See [Table 1] No. 1~No.16

Input parameters affect the stitching speed. -See [Table1] No.1~No.16

Therefore, the setting of the auto stitching parameter according to the image characteristics is important for stitching performance.
