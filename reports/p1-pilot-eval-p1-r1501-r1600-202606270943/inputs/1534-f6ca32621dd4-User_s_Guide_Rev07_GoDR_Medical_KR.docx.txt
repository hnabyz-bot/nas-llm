제품 소개

GoDR은 병원 정보 시스템으로부터 검사해야 할 환자의 목록을 가져오거나 수동으로 검사 할 환자의 정보를 입력하여 영상을 획득, 저장한 후, PACS(Picture Archiving and Communication System)로 전송하는 역할을 담당하는 소프트웨어입니다.

주요기능

환자 정보와 이미지 정보의 관리

환자 영상의 진단 및 분석

치료계획 수립을 위한 시뮬레이션

사용목적

획득된 의료용 영상을 저장, 조회, 확대, 축소와 함께 분석, 전송 처리하는 의료용 소프트웨어입니다.

사용환경 요구사항

OS : (한글/영문) Microsoft Windows 10 with Net Framework 4.5.2

CPU : Intel 5 이상

RAM : 4GB

HDD : 500GB

Graphic Card : Intel HD2500 이상

모니터 : 해상도 1920 X 1080 이상 표현 가능한 모니터

최종 소프트웨어 버전

Version 1.1.28.0

실행하기

바탕화면에 생성된 GoDR 아이콘을 더블 클릭하여 GoDR를 실행합니다.

정식 라이선스로 사용하거나, Demo 버전으로 사용할 수 있습니다.

Try Demo 선택시 GoDR을 기능 제한 없이 바로 사용할 수 있으며, 기간은 최대 10일 입니다.

Register 선택시 라이선스 입력창이 뜨게되며 정식 발급 받은 라이선스를 입력하고 ‘OK’ 버튼을 클릭하면 GoDR 이 실행됩니다.

*정식 라이선스는 구입처에 문의하시기 바랍니다.

로그인 하기

GoDR이 실행되면 로그인 화면이 나타납니다.

로그인할 사용자를 선택하고 ‘로그인’ 버튼을 클릭하여 로그인합니다. RAD 계정의 초기 비밀번호는 “rad” 입니다. 설치 후, 반드시 암호를 변경해 주시길 바랍니다. 암호 변경은 [스마트온버튼 - 설정보드 - 사용자] 란에서 변경 가능합니다.

로그인 후에는 워크리스트보드로 진입하게 됩니다.

<워크리스트보드>

설정보드( 워크리스트보드 좌측 상단 아이콘 선택->설정)

*설정보드에서 내용 변경 시 좌측 하단의 저장 버튼을 반드시 눌러야지만 변경사항이 저장 됩니다.

사용자 : 사용자 이미지, 사용자 아이디, 비밀번호를 변경할 수 있습니다.

비밀번호 변경하는 방법

 해당 버튼을 클릭합니다.

비밀번호 란에 현재 비밀번호를 입력하고 새 비밀번호, 비밀번호 확인란에 변경할 비밀번호를

입력한 뒤 버튼을 클릭합니다.

사용자 추가가 필요한 경우, 서비스엔지니어에게 문의하시기 바랍니다.

영상정보 표시 / 마킹

영상정보표시 / 마킹 탭에서는 디시전보드에서 보여주고자 하는 영상정보를 선택하여 설정할 수 있으며, 마킹툴과 관련된 설정 작업을 할 수 있습니다.

영상정보표시

선택 또는 선택해제하여 디시전 보드에 띄울 정보를 설정할 수 있습니다. 선택한 정보는 디시전보드에서 영상 좌측에 표시됩니다.

DisplayInfosOnImage

 선택 또는 선택해제하여 디시전보드의 영상에 선택한 DICOM 정보를 넣어줍니다. 선택시 나타나는 옵션에서 위치 및 글씨 크기 설정이 가능합니다. 설정을 하면 영상 위에 영상 정보가 표시됩니다.

<영상정보표시 / DisplayInfosOnImage >

마킹

입력창에 마킹에서 사용할 단어를 입력하고 버튼을 클릭해서 단어를 추가합니다.

선택 또는 선택 해제를 통하여 디시전보드의 마킹 리스트에서 보여질 마킹을 설정할 수 있습니다.

 버튼을 사용하여 사용하지 않을 마킹을 삭제할 수 있습니다.

세팅이 완료되면 버튼을 클릭하여 적용시킵니다.( 버튼을 클릭하지 않으면 적용이 안됩니다.)

환자정보가져오기

검진용 차량과 같이 장비가 외부에서 사용되는 경우를 고려해서 설계된 기능입니다.

 : 워크리스트에서 환자목록을 보여줄 형식을 선택합니다.

체크된 상태(기본값)일 경우 리스트형식, 체크가 해제된 상태일 경우 카드형식으로 보여집니다.

검사리스트 구성

 : 검사리스트를 먼저 설정한 후 환자리스트를 가져옵니다.

아래와 같이 검사리스트를 설정할 수 있는 창이 뜹니다.

인체도 영역에서 머리부분을 클릭하거나 MainBodyPart 에서 ‘Skull’ 버튼을 클릭하고, BodyPart 에서 ‘Mandible’, Projection 에서 ‘AP’ 를 선택하면 검사리스트가 생성됩니다. 검사리스트의 괄호 안에는 Detector, Parameter 값이 정의 되어있습니다.

 버튼을 클릭합니다. (여기까지 검사리스트 생성)

 : 검사리스트의 구성 없이 환자리스트를 불러올 수 있습니다. 방법은 [ 3)-a ~ 3)-d ] 와 같습니다.

파일열기 다이얼로그 창이 뜨면 csv, txt 파일로 구성된 불러올 환자리스트가 저장되어있는 경로를 찾아 해당 파일을 ‘열기(O)’합니다.

 이와 같이 불러온 환자의 수를 확인 할 수 있으며, 버튼을 클릭하여 워크리스트에 불러온 환자의 리스트를 적용합니다.

 : 추가되어 있는 검사 리스트를 삭제할 수 있습니다.

아래와 같이 워크리스트에서 적용된 환자 리스트를 확인할 수 있습니다.

디스크정보

영상 저장위치 : 획득한 영상이 저장되는 위치를 설정합니다. ‘경로선택’ 버튼을 사용하여 저장 위치를 변경할 수 있습니다. (가급적이면 기본 설정 값을 유지하는 것을 권장합니다.)

보관일수 : 프로그램 종료 시, 입력한 기간을 제외한 데이터와 영상을 삭제합니다.

수동으로 강제 삭제 :

디스크 사용량 상한 값 : 프로그램이 사용 가능한 디스크 용량의 상한선을 설정합니다.

디스크 사용량 하한 값 : 프로그램이 사용하는 디스크 용량이 상한선에 이르면 확보할 디스크의 용량의 하한선을 설정합니다.

위의 그림처럼 상한선 90%, 하한선 60% 로 설정되어 있을 경우, 디스크 사용량이 전체 디스크 용량의 지정 상한선(90%)에 이르면 디스크의 용량을 확보하기 위해 지정된 하한선(60%)까지 가장 오래된 검사정보부터 자동으로 삭제합니다.

세팅이 완료되면 버튼을 클릭하여 적용시킵니다.( 버튼을 클릭하지 않으면 적용이 안됩니다.)

검사맵핑

병원의 검사코드를 실제의 검사정보와 손쉽게 Mapping할 수 있도록 설정할 수 있는 화면입니다.

 : 검색기능

입력창에 찾고자 하는 검색어를 입력한 후에 ‘찾기( )’ 버튼을 클릭하여 찾고자 하는 오더맵핑 정보를 검색할 수 있습니다.

 체크박스에 체크표시를 할 경우 Hide 된 오더맵핑을 확인할 수 있습니다. Hide된 오더맵핑은 워크리스트에 표시되지 않습니다.

 : 매핑정보 내보내기

오더맵핑 정보를 *.csv 파일로 저장합니다.

 : 매핑정보 가져오기

*.csv 파일로 구성된 오더맵핑 정보를 불러옵니다.

 : 새로운 오더맵핑정보를 추가할 수 있습니다.

 버튼을 클릭하면 아래와 같이 오더맵핑정보를 추가할 수 있는 다이얼로그 창이 뜹니다.

알림바 & 스마트온버튼 & 스마트온보드

알림바

현재 로그인되어 있는 방사선사의 아이디를 비롯하여 장비들의 연동 상태를 확인할 수 있으며, 상태 표시줄의 색상으로 현재 작업 중인 화면의 구분이 가능합니다.

알림바의 색상

 : 장비연동이 정상적으로 되어있는 상태

 : 장비연동이 되지 않은 상태

 : 촬영준비가 된 상태(DR 디텍터 레디 상태)

기타 정보

로그인한 방사선사의 아이디를 나타냅니다.

현재 날짜 및 시간을 보여줍니다.

1번 디텍터를 레디 시킵니다. (선택 시 흰색으로 표시)

2번 디텍터를 레디 시킵니다. (선택 시 흰색으로 표시)

무선으로 연결이 되었는 디텍터를 레디 시킵니다. (선택 시 흰색으로 표시)

연결되어 있는 DR 디텍터를 사용하지 않고 X-ray를 사용할 경우 선택합니다. (ex. CR사용 등) (선택 시 흰색으로 표시)

무선디텍터의 경우 무선디텍터 배터리의 잔량을 나타냅니다.(유선일 경우 나타나지 않음)

무선디텍터의 경우 WiFi 세기를 나타냅니다.(유선일 경우 나타나지 않음)

설정보드에서 지정한 저장소의 남은 저장 용량을 표시합니다.

워크리스트 서버 연결 상태를 표시합니다. (서버와 연결이 되어 있지 않을 경우 회색으로 나타나며, 연결이 되어 있을 경우 흰색으로 나타납니다.)

스토리지 서버 연결 상태를 표시합니다. (서버와 연결이 되어 있지 않을 경우 회색으로 나타나며, 연결이 되어 있을 경우 흰색으로 나타납니다.)

프린터 서버의 연결 상태를 표시합니다. (서버와 연결이 되어 있지 않을 경우 회색으로 나타나며, 연결이 되어 있을 경우 흰색으로 나타납니다.)

스마트온 버튼

 : 어느 화면에서도 알림바의 ‘스마트온 버튼’을 클릭하면 ‘스마트온보드’로 이동이 가능합니다. (디시전보드에서 디텍터 연동이 안되었을 시 제외)

스마트온보드

 워크리스트 : 표시된 숫자는 Scheduled List 의 수를 의미하며 클릭시 Scheduled List 화면으로 이동합니다.

 스코어 보드 : 오늘 획득한 영상의 수와 최대 처리 영상(스캔 및 임포트)의 수를 확인 할 수 있습니다.

 Quick Button

 워크리스트 : Scheduled List 화면으로 이동합니다.

 새로운 검사생성 : 언제든지 ‘스마트온보드’에서 신규 검사를 수동으로 생성할 수 있습니다.

 로그오프 : 사용자 변경을 위하여 로그오프합니다.

 대기목록 : Pending List 화면으로 이동합니다.

 실패목록 : Error List 화면으로 이동합니다.

 완료목록 : Success List 화면으로 이동합니다.

 디스크 상태정보 : 현재 로컬 저장소의 사용 상태를 확인할 수 있습니다.

 네트워크 전송상태

DICOM Storage : 오늘 전송 실패 수 / 오늘 전송 시도 수

DICOM Print : 오늘 프린트 실패 수 / 오늘 프린트 시도 수

 영상현황판 : 각 요일 별 처리 영상 수를 확인 할 수 있으며, 처리한 영상 수(누적)를 확인할 수 있습니다. (DR 스캔수 + 임포트 영상 수 포함)

 이전으로 : 스마트온보드 접근 전의 화면으로 되돌아갑니다.

 설정 : 설정보드에 진입합니다.

 시스템종료 : 프로그램을 종료합니다.

워크리스트보드(스케줄리스트)

 : 워크리스트보드의 검사 수(카드 수, 리스트 수)를 카운트합니다.

필터링 패널

검사 대기중인 환자 목록을 표시합니다. (숫자는 리스트의 개수를 의미합니다.)
