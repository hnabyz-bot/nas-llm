1. Introduction
feel-DRCS is the name of the digital radiography of imfoU Co. Ltd. feel-
DRCS is the DR console or DR control software that acquires image from the
DR detector like flat panel or CCD, processes rapidly for clinical application,
and provides DICOM services for the integration with PACS system. This
manual explains how to install and operate feel-DRCS software.

2. Purpose
feel-DRCS is the operation console system software integrated with various
detectors or X-ray generator, based on FS-MLW (Faster Specialized Multi
Layered Wavelet) technology.
feel-DRCS is the product developed for the operator, designed at the clinical
hospital through many years of clinical tests, and finished by service
engineers of former major DR makers.
feel-DRCS is the embedded software based on feel-TK enable to remove
noise and various artifacts and much clearer body tissue. The embedded
calibration function makes it possible to set the basic calibration data (offset
image and gain image) for all types of detector. It can also process
progressive defect by providing defect view.
feel-DRCS is distributed and being used at the clinical fields after verifying
software on various detectors distributed worldwide. It is easy to use console
system’s software just by changing the type of the detector which DR makers
want. Integrating the X-ray’s generator operation panel, the total DR system
inside the software makes it possible to control APR, AEC, Ready/Exposure
with ease.
feel-DRCS is integrated with various X-ray generators, supports APR function,
and provides all-in-one DR product which enables to work as existing
operation panel with just one console software.
feel-DRCS does not provide the software the DR makers, but the
knowhow as well as consulting services to be your best partner.
2-1. Intended use
Feel-DRCS is intended to be used for radiological doctor and radiological
technician for acquired general radiographic images.
So, it’s used by radiological doctor or radiological technician in hospital.

3. System requirements
3-1. Workstation
 Processor: Intel Core i3 or later(recommend : i5 or more)
 RAM : 4GB RAM(Recommend 8GB)
 HDD: At least 100GB hard disk. For data backup and reliability, we
recommend two separate HDDs physically-1st HDD: System and
Software, 2 HDD: Image Data Storage, not separating the partition
logically with just one HDD.
 Network Card : 10/100/1000Mbps Ethernet Network Card
 Video graphic adapter: at least over 1280 X 1024 pixels, color
resolution-True Color mode(no shard memory), DVI interface connector
 Operating System : Windows 7 prof.(at least SP1), Windows 10
 Need I/O : 1 Parallel port(synchronize Detector & X-ray Generator &
Software), 1 Serial port(in case Control of X-ray generator), 1 USB
port(Software Lock-Key)
3-2. Monitor
 Over 19 inch (23 inch recommended)
 Brightness : over 300cd/m2
 Contrast Ratio : over DC 8000:1
 Response time : over 5 ms
 1600X1200 or 1920X1280 recommended (at least over 1280 X 1024) 4:
3 , 16:9. 16: 10 ratio supported.
 High fidelity of grayscale and optimized luminance distribution
If you want to use high-resolution monitor, we recommend medical LCD
monitor (B/W or color). There is no problem because the present software
was made based on gray color when you want to use black and white
monitor.
For image quality control, we recommend the below among medical
monitors. It is because it needs to adjust the image at the same brightness
for diagnosis after taking X-ray. Most of monitors are made according to
each country’s medical standard so that there is no problem to use them.

1) System access control and authentication, connection recognition, connection restriction
 All users must set their ID and password to use.
 The system administrator should manage to ensure that all users maintain a valid a
ccount, especially when moving or leaving work.
2) Managing User Authentication
 The system administrator should set and manage the expiration date of the accoun
t for each user, check the expiration period for each cycle, and control access whe
n the expiration date is set.
3) Automatic session termination
 The system administrator should set up the power management function of the Wi
ndows on the computer where Feel-DRCS is installed so that communication and co
nnection of the system are terminated after the set time.
4) Password Creation Rules
 Passwords must meet the "technical and administrative protection criteria for perso
nal information".
 At least 10 digits in combination of two or more types of English, numbers, and sp
ecial characters, or at least 8 digits in combination of three types.
 Do not use numbers similar to personal information and IDs that are easy to guess,
such as consecutive numbers, birthdays, and phone numbers.
 Make sure to change the password at least once a half-year.
5) Authorization of software updates
 Without a "License" granted by the manufacturer, the user cannot update the soft
ware.
 Software updates are made by visiting our customer support team.
