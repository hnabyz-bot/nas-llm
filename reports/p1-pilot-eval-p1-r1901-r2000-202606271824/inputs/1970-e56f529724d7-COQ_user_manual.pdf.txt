COQ: an ImageJ plugin for the physical characterization
and quality checks of digital detectors.
This software was developed for assisting users in achieving the physical characterization
of X-ray digital systems and some image quality checks. It can be used for calculating
various physical parameters such as the response curve, Modulation Transfer Function
(MTF), Noise Power Spectra (NPS), Detective Quantum Efficiency (DQE). It also includes
the computation of some image quality checks: defective pixel analysis, uniformity, dark
analysis, and lag.

1. Installation
If ImageJ is not installed in your computer, download it from
http://imagej.nih.gov/ij/download.html and install it following the instructions.
If you have already installed an old version of ImageJ (version 1.43 or older) in your
computer, you need to upgrade ImageJ to the latest version. You can upgrade ImageJ, by
downloading and installing the latest version from http://imagej.nih.gov/ij/download.html.
If you have already installed ImageJ (version 1.44 or later) in your computer, you just need
to download the plugin, without the need of upgrading ImageJ.
Download COQ_jar from http://www.medphys.it/down_dqe.htm
Place COQ_.jar in the plugins folder of ImageJ.
1.1. Technical requirements
For a correct functioning of the COQ plugin the following requirements should be satisfied:
• ImageJ installed (version 1.44 or later).
• Screen resolution: at least 1280×800.

6. Control panel
• Response curve
i. Purpose: to calculate the relationship between detector dose and pixel value
in a range compatible with clinical conditions
ii. Procedure:
1. Open (using “Open” button) the images you acquired (at least 4 images to
have an accurate fit).
2. Place a ROI of about 2 cm x 2 cm in a central position of the images.
3. Click “Calculate” button and a new window will open.
4. For each image insert the Air Kerma in μGy and choose the curve type
between:
a. Linear:
[y = a + b·x]
b. Logarithmic 2:
[y = a + b·log(1+c·x)]
c. Logarithmic 3:
[y = a + b·log(x)]
d. Square Root:
[y= a + b·sqrt(x)]
5. Then click OK

5/10
6. In Result panel will be shown the fitting results (parameters a,b, (c) and
the squared correlation coefficient R2) of the Response curve (or
Conversion function).
7. Using “Export” button results can be saved in a *.csv file.
8. From now on, the “Custom” radio button will be selected, the selected
fitting function is reported and all the images will be linearized in dose.
• DQE (Detective Quantum Efficiency)
i. Purpose: to calculate detector quantum efficiency measuring the Modulation
Transfer Function (MTF) and the Noise Power Spectrum (NNPS)
ii. Procedure
1. MTF
a. Vert. Edge (Vertical Edge): the procedure is formally identical as for
Hor. Edge.
b. Hor. Edge (Horizontal Edge). Select a rectangular ROI along the edge
(i.e.: yellow ROI in the following figure). This ROI selects only the length
along the edge. The profile length in subpixel size (defined as the pixel
pitch in mm divided by 10) will be selected within the MTF new window
(pop-up menu).

6/10
c. Ver. Slit (Vertical Slit)
d. Hor. Slit (Horizontal Slit): Select a rectangular ROI along the slit (i.e.:
yellow ROI in the following figure). This ROI must be positioned, in
order to select the area of the image where the slit is located. Check if
the pixel size (in mm) shown in the MTF new window (pop-up menu) is
correct, otherwise write the correct value in the text box.
2. NNPS
a. Open the image/s to analyze and select a ROI (i.e.: 1024 × 1024 pixels)

7/10
b. Click “NNPS” button and a new window will be opened:
i. Check that “Pixel size (mm)” is correct
ii. “NNPS fragment size” is the size of the running subROI
iii. “Number of lines to omit for NNPS” is the number of lines in the
proximity of the principal axes that will not be considered for NNPS
estimation
iv. “Number of lines to use for NNPS” is the number of lines near the
axes (starting from the first line not excluded, see point iii) that will
be considered for NNPS estimation
v. Enabling “Show 2D NNPS” a 2D representation is shown in the
plot panel (in the left)

8/10
3. Compute DQE
a. Select the right “X-RAY BEAM” (RQA 3, RQA 5, RQA 7, and RQA 9 are
yet tabulated and for those beams SNR2in is automatically compiled,
otherwise it should be inserted manually)
b. Select the Air Kerma in μGy
c. Click “OK” button to calculate DQE.
• Bad Pixel
i. Purpose: to analyze an image uniformly irradiated to check the accuracy of the
defective pixel/lines correction in the preprocessing stage.
ii. Procedure:
1. Open the image to analyze
2. Click “Calculate” button to analyze the image within 10 mm × 10 mm ROIs. A
defective detector element is an element of the ROI where the absolute
value deviates more than 20% from the mean pixel value of the ROI
elements.
3. The number of bad pixels and their location (x,y coordinates) are calculated
and displayed in the “Result” panel.

9/10
• Uniformity
i. Purpose: to test the flat field correction by evaluating a uniform flat image over
the full area of the detector
ii. Procedure:
1. Open the image to analyze
2. Click “Calculate” button to analyze the image within 30 mm × 30 mm ROIs
overlapped by ½ ROI.
3. Local Signal Non-Uniformity (LSNU), Global Signal Non-Uniformity (GSNU),
Local SNR Non-Uniformity (LSNRNU), and Global SNR Non-Uniformity
(GSNRNU) are calculated and displayed in the “Result” panel.
4. The Reference limits for LSNU, GSNU, LSNRNU, and GSNRNU are
respectively: 4%, 8%, 8%, and 20%.
• Dark analysis
i. Purpose: to assess the intensity of intrinsic noise in the system
ii. Procedure:
1. Open the image to analyze
2. Click “Calculate” button to analyze the image within 10 mm × 10 mm ROIs. A
defective detector element is an element of the ROI where the absolute value
deviates more than 20% from the mean pixel value of the ROI elements.
• Lag
i. Purpose: to evaluate the severity of any artifact due to previous exposure to the
detector.
ii. Procedure:
1. For additive lag open two images: first an irradiated image with a test
phantom (e.g., the edge used for the MTF calculation) and then a
nonirradiated image. For multiplicative lag open three images: first an image
irradiated with a test phantom, and then two uniform images with no
phantom, one acquired before, and one after the irradiated test image. Press
“Continue”.
2. Move the first ROI within the phantom and press “Continue”.
3. Move the second ROI outside the phantom and press “Continue”.
• AGD/CNR (for mammography only)
i. Purpose: to estimate the AGD and the contrast-to-noise ratio (CNR), in order to
correlate image quality to patient dose in mammography.
ii. Procedure:
1. Select the images (at least 2) with different PMMA thicknesses.
For AGD:
2. Select whether to use the ESAK data estimated from the tabulated data or
the entrance dose extracted from the DICOM file.
Note: if one wants to modify the data for calculating the AGD, set the “Compute
using external file” option to “On” from the “Config” panel, and then provide a .csv
file containing the two tables for “HVL” and “Tube output” (see for example the
Data_AGD.csv file present in the coq.zip archive).
For CNR:
2. Select the number and size of the ROIs to be used.
3. Move the ROI on the right (when 2 ROIs are used) or on the center (when 5
ROIs are used) within the phantom and press “Continue”.
4. Repeat point 3 for all the images.
